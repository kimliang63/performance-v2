## 一、背景与目标

### 1.1 业务背景

HRONE 绩效管理系统是企业人力资源管理的核心模块，覆盖绩效目标制定、考核评分、结果审定、员工确认、绩效面谈的完整生命周期。系统采用三层配置架构：

- **V1 基础配置层**（SSC 或系统配置管理员维护）：绩效周期、单位、等级规则、计算规则、衡量标准、指标库、评价规则、强制分布规则 — 已评审锁定
- **V2 方案配置层**（SSC 管理）：绩效方案，定义考核组、考核环节、流程关联、表单关联 — 本 PRD 范围
- **V4 活动执行层**（BP 按权限处理部分业务数据）：考核活动，基于方案模板实例化执行 — 下一版本，不在本 PRD 方案管理范围内

### 1.2 当前问题

1. 旧系统（OEHR）绩效流程高度硬编码，无法灵活配置不同考核场景（月度/季度/半年度/年度）
2. 考核组无法按条件（部门/职位/职级）自动匹配人员，依赖手工维护
3. 强制分布规则与等级规则分离，无法在方案层面灵活配置
4. 流程模型、表单模板、考核环节之间的关联关系缺乏统一管理
5. 方案配置修改后影响范围不明确，缺乏版本管理和快照机制

### 1.3 本版本目标

1. 支持 SSC 创建/编辑/复制/启用/停用绩效方案
2. 支持多考核组配置，按条件（部门/职位/职级）+ 人员交集确定考核范围，优先级分配
3. 支持向导式配置：基本信息 → 考核组 → 考核环节设计（目标制定/绩效考核/结果审定/结果确认/绩效面谈）
4. 各考核环节可独立配置开启/结束方式、流程模板、通知、强制分布参数
5. 方案停用后已创建的活动不受影响（快照隔离机制）
6. 提供预置表单模板管理能力

### 1.4 不在本版本范围内的内容

| 内容 | 说明 |
|------|------|
| V1 基础配置 | 已评审锁定，详见 prd-v1-basic-config.md |
| V3 考核表单与流程 | 表单字段定义和流程模型定义，详见 prd-v3-考核表单与流程.md |
| V4 考核活动管理 | 活动生命周期、参与人匹配、阶段管控，下一版本 |
| V5 员工工作台 | 员工端 PC/移动端，预留版本 |
| V6 AI 应用 | AI 表单助手、AI 监控、AI 分析，原型完成待集成 |
| 绩效申诉模块 | V2 仅配置结果确认环节及流程模板；员工不同意、BP 知悉、线下处理属于 V4 执行参考，不在本版本落地 |
| 消息通知模板 | 由消息通知功能预置，本版本仅配置通知类型×渠道 |
| 角色权限管理 | 由组织人事模块维护，绩效不单独维护 |
| 预置表单模板 | 表单模板在流程引擎中配置和管理，非绩效方案功能 |

## 二、角色与使用场景

### 2.1 使用角色

| 角色 | 职责 | 在本 PRD 中的操作 |
|------|------|------------------|
| SSC | 统一管理绩效方案配置 | 创建、编辑、复制、启用、停用、逻辑删除方案；配置考核组、考核环节、流程模板、规则引用、预置表单 |
| 系统管理员/HRIS | 维护系统权限、组织架构、基础配置支撑 | 不直接管理绩效方案；保障 SSC 可使用所需基础数据和权限 |
| BP | 负责活动执行过程中的部分业务数据处理 | 不参与方案管理；V4 活动执行中另行定义其操作范围 |
| 员工 | 参与绩效执行流程 | 不参与方案管理；员工端能力在 V5 工作台范围 |

### 2.2 典型业务场景

**场景一：SSC 创建月度绩效方案**
1. SSC 进入绩效方案列表页，点击"新增"
2. 后端创建草稿方案记录，生成方案编码和版本号，系统进入向导式配置
3. 使用月度绩效模板（默认开启：目标制定→绩效考核→结果审定→结果确认）
4. SSC 填写基本信息（名称、周期类型=月度、适用部门、等级规则、动态考核方式，"是否启用"选择"启用"）
5. SSC 配置考核组（按部门条件匹配销售部员工，评价规则选择数值打分）
6. SSC 为每个环节关联流程模板，配置强制分布参数
7. SSC 点击"保存"，系统全量校验通过，方案状态按"是否启用"字段变为"已启用"

**场景二：SSC 创建年度绩效方案（含面谈）**
1. SSC 选择年度绩效模板，系统自动开启：绩效面谈→结果审定
2. SSC 需手动开启目标制定、绩效考核、结果确认环节
3. SSC 配置绩效面谈环节的流程模板
4. 面谈环节必须在结果确认之前完成（系统强制校验）

**场景三：SSC 编辑已启用方案**
1. SSC 在列表页点击已启用方案的"编辑"按钮
2. 系统跳转向导页，回填已有配置
3. SSC 修改环节配置（如调整强制分布参数）
4. SSC 点击"保存"，系统全量校验通过，版本号+1，方案保持"已启用"状态
5. 已创建的活动不受影响（活动使用创建时的快照配置）

**场景四：SSC 停用方案**
1. SSC 在列表页点击已启用方案的"停用"按钮
2. 系统检查：该方案关联的进行中活动数=0 → 允许停用；>0 → 阻断，提示"当前方案存在进行中的绩效活动，不可停用"
3. 停用后，方案状态变为"已停用"，不可发起新活动
4. 已创建的活动继续按活动快照配置执行

> 后续 V4 活动执行模块在创建活动时，只能选择已启用且未逻辑删除的方案。

### 2.3 端到端业务流程概览

```
SSC 创建方案（草稿）
    ↓ 保存（启用=是）
方案（已启用）
    ↓ V4 活动执行（BP 按权限处理业务数据）
活动实例（快照方案配置）
    ↓ 各环节执行
目标制定 → 绩效考核 → 结果审定 → 绩效面谈（可选） → 结果确认
    ↓
绩效结果锁定
```

## 四、详细功能需求

### 4.1 绩效方案列表

#### 功能目标
为 SSC 提供绩效方案的统一管理入口，支持方案的查看、创建、编辑、复制、启用、停用、删除操作。

#### 适用角色
SSC

#### 前置条件
用户拥有"绩效方案-查询"权限

#### 页面入口
HRONE / 绩效管理 / 绩效方案 / 绩效方案

#### 用户操作流程
1. 用户进入方案列表页
2. 页面展示方案列表（默认每页20条，按最近修改时间倒序）
3. 用户可通过搜索框按方案名称模糊搜索，或通过状态下拉筛选
4. 用户根据方案状态执行对应操作（编辑/复制/启用/停用/删除）

#### 前端交互逻辑
- **页面加载**：请求后端获取方案列表数据，同时获取统计卡片数据（总数/已启用/已停用/草稿数量）
- **搜索**：输入关键词后按回车或点击搜索按钮，触发列表刷新；清空关键词恢复默认列表
- **筛选**：状态下拉选择后立即触发列表刷新
- **操作按钮**：根据方案状态动态展示，不满足条件的按钮置灰或隐藏
- **分页**：每页20条，支持翻页
- **新增按钮**：点击后后端创建草稿方案记录（生成编码和版本号），进入方案配置向导（步骤1-基本信息）

#### 字段说明

| 字段     | 类型   | 说明                       |
| ------ | ---- | ------------------------ |
| 方案名称   | 文本   | 可点击，跳转方案详情页              |
| 编号     | 文本   | 系统自动生成（技术定义生成规则），不可编辑              |
| 版本     | 数字   | 显示当前版本号                  |
| 状态     | 标签   | 草稿（灰色）/ 已启用（绿色）/ 已停用（红色） |
| 向下公开   | 是/否  | 基于关联的组织架构，控制该组织架构下的员工是否可在工作台查看该方案关联的绩效结果 |
| 最近修改人  | 文本   | 最近修改的操作人                 |
| 最近修改时间 | 日期时间 | 格式 YYYY/MM/DD HH:mm:ss   |

#### 列表操作

| 操作  | 可用状态       | 前端行为                      | 后端处理                                              |
| --- | ---------- | ------------------------- | ------------------------------------------------- |
| 编辑  | 草稿、已启用、已停用 | 跳转向导页，回填已有配置              | 查询方案完整配置                                          |
| 复制  | 所有状态       | 弹出二次确认"是否确认复制该方案？"        | 生成新编号，版本重置为1，状态为草稿，复制全部配置                         |
| 启用  | 草稿、已停用     | 弹出二次确认"是否确认启用该方案？"        | 执行完整校验（基本信息+考核组+环节+引用配置可用性）→ 通过则状态变为已启用；不通过展示失败汇总 |
| 停用  | 已启用        | 弹出二次确认                    | 检查关联进行中活动数=0 → 通过则状态变为已停用；>0 阻断                   |
| 删除  | 草稿、已停用     | 弹出二次确认"是否确认删除该方案？删除后不可恢复" | 逻辑删除（方案不在新建活动时可见、不可被选择；历史活动仍保留对该方案的快照展示）          |

#### 后端校验规则

| 校验项    | 规则                                                                            | 错误提示                      |
| ------ | ----------------------------------------------------------------------------- | ------------------------- |
| 启用校验   | 基本信息必填项完整 + 至少1个考核组且必填项完整 + 至少1个环节开启且流程模板已关联 + 引用配置（等级规则、评价规则、流程模板、强制分布规则）均可用 | 按步骤展示失败汇总                 |
| 重新启用校验 | 与启用校验相同：完整校验所有必填项和引用配置可用性                                                     | 按步骤展示失败汇总                 |
| 停用校验   | 关联进行中活动数=0                                                                    | "当前方案存在进行中的绩效活动，不可停用"     |
| 删除校验   | 状态为草稿或已停用                                                                     | "当前状态不可删除"；已启用方案不可删除，需先停用 |
| 删除后果   | 逻辑删除后方案不可恢复                                                                   | "删除后不可恢复，确认删除？"           |
| 名称唯一性  | 同语种下方案名称不重复                                                                   | "方案名称已存在，请修改"             |

#### 异常情况及处理规则

| 异常场景 | 用户可见提示 | 系统处理方式 |
|---------|-------------|-------------|
| 网络请求失败 | "网络异常，请稍后重试" | 保持当前页面状态，不丢失已输入内容 |
| 方案被其他用户同时编辑 | "方案已被其他用户修改，请刷新后重试" | 刷新列表数据 |
| 启用校验失败 | 弹窗展示校验失败汇总，支持跳转到失败步骤 | 保持草稿状态 |

#### 权限与可见性规则
- 无查询权限：显示无权限缺省页
- 无新增权限：新增按钮隐藏
- 无编辑权限：编辑按钮隐藏或置灰
- 无启用/停用权限：对应按钮隐藏或置灰

### 4.2 方案配置向导

#### 功能目标
通过向导式交互引导 SSC 完成绩效方案的完整配置，支持步骤切换暂存、全量校验保存、保存草稿三种保存模式。所有保存操作均提交到后端。

#### 适用角色
SSC

#### 前置条件
用户拥有"绩效方案-新增"或"绩效方案-编辑"权限

#### 页面入口
方案列表页点击"新增"或"编辑"

#### 用户操作流程
1. 用户点击"新增"，后端创建草稿记录并生成编码，进入向导页，默认展示步骤1-基本信息
2. 左侧目录展示配置步骤和完成状态
3. 用户按顺序或跳转填写各步骤内容，切换步骤时当前步骤内容自动提交到后端暂存
4. 用户可点击"保存草稿"暂存当前步骤，或点击"保存"全量校验并保存

#### 前端交互逻辑

**目录行为**：
- 步骤包含：基本信息、考核组、考核环节设计（含子目录：目标制定、绩效考核、结果审定、结果确认、绩效面谈）
- 默认使用月度绩效模板
- 每一步骤条包含3个状态图标：未配置（灰色空心圆）、配置中（蓝色实心圆）、已完成（绿色对勾）
- 配置进度：展示"配置完成度"进度环（百分比），计算口径 = 已完成节点数 / 总节点数 × 100%，向下取整
- 收起/展开：点击收起按钮可折叠目录区域，折叠后保留当前激活节点标识
- 点击任意节点可跳转到对应节点

**模板设置**：
- 五个考核环节（目标制定、绩效考核、结果审定、结果确认、绩效面谈）固定不可增删
- 包含模板：月度绩效、半年度绩效、年度绩效
- 默认使用月度绩效
- 模板与环节默认开启：

| 模板 | 默认开启环节 | 顺序 |
|------|-------------|------|
| 月度绩效 | 目标制定、绩效考核、结果审定、结果确认 | 目标制定→绩效考核→结果审定→结果确认 |
| 半年度绩效 | 绩效面谈 | — |

- SSC 在模板基础上可手动勾选/取消勾选环节，但环节本身不可增删

### 4.3 基本信息

#### 功能目标
配置方案的基础属性，包括名称、周期类型、适用部门、等级规则、动态考核方式等。

#### 适用角色
SSC

#### 前置条件
处于方案配置向导-步骤1

#### 用户操作流程
1. 用户进入基本信息步骤
2. 填写各字段（名称、周期类型、部门、等级规则等）
3. 切换步骤时当前步骤内容提交到后端暂存

#### 前端交互逻辑
- **名称**：必填，单行文本框，默认为空，提示"请输入"；最多可输入200个字；点击多语言按钮弹出多语言编辑器
- **编号**：系统自动生成，不可编辑；新增时按规则生成唯一编码，编辑时不更新
- **周期类型**：必填，单选下拉，默认为空；选项值：月度、季度、半年度、年度、自定义
- **部门**：必填，单选，选择方案适用的组织范围；选项值获取组织架构树数据
- **是否启用**：必填，单选，默认选中"启用"；选项值：启用、停用。控制保存后方案的最终状态：选择"启用"则保存后方案为已启用状态；选择"停用"则保存后方案为已停用状态
- **向下公开**：必填，单选，默认选中"是"；选项值：是、否。基于关联的组织架构，控制该组织架构下的员工是否可在工作台查看该方案关联的绩效结果
- **动态考核**：必填，单选，默认为空；选项值：开启时锁定考核关系、动态获取考核关系
- **绩效等级**：必填，单选，默认为空；选项值获取【等级规则】中当前已启用的等级信息
- **规则描述**：非必填，多行文本框，默认为空，提示"请输入"；最多可输入500个字

#### 字段说明

| 字段 | 字段含义 | 输入/展示/系统生成 | 是否必填 | 默认值 | 校验规则 | 可编辑条件 |

| 名称 | 方案名称 | 输入 | 是 | 空 | 不为空，≤200字，同语种唯一 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 编号 | 方案唯一标识 | 系统生成 | — | 自动生成 | 唯一性 | 不可编辑 |

| 是否启用 | 保存时选择方案状态 | 输入 | 是 | 启用 | — | 新建、草稿、已启用、已停用重新编辑时均可编辑 |

### 4.4 考核组配置

#### 功能目标
配置绩效方案的考核组，定义考核范围（按条件或按人员），支持多组优先级分配。

#### 适用角色
SSC

#### 前置条件
处于方案配置向导-步骤2

#### 用户操作流程
1. 用户进入考核组步骤
2. 系统默认展示一项"默认考核组"
3. 用户可新增考核组、编辑现有考核组、删除手动创建的考核组
4. 每个考核组选择模式（按条件/按人员），配置对应规则

#### 前端交互逻辑
- **考核组模式**：必填，单选，默认为空，提示"请选择"；选项值：按条件选择、按人员选择
- **考核组列表**：
  - 展示当前所有考核组信息：名称、优先级、规则摘要
  - 操作：新增（弹出新增考核组弹窗）、编辑（弹出编辑考核组弹窗，默认考核组不可编辑）、删除（仅可删除手动创建的考核组，默认考核组不可删除）
- **新增/编辑考核组弹窗**：
  - 名称：必填，单行文本框，最多200字，支持多语言
  - 优先级：必填，单行文本框，仅可输入正整数，数字越小优先级越高
  - 评价规则：必填，单选，选项值获取【评价规则】中当前已启用的规则
  - 按条件选择模式：展示条件规则区域（条件类型：部门/职位/职级/划分区域/人员类型/人员状态/手动选人 + 条件值 + 排除人员）
  - 按人员选择模式：展示人员选择框

#### 字段说明

| 字段 | 字段含义 | 输入/展示/系统生成 | 是否必填 | 默认值 | 校验规则 | 可编辑条件 |
|------|---------|-------------------|---------|--------|---------|-----------|
| 考核组名称 | 考核组标识 | 输入 | 是 | "默认考核组"（默认组）/ 空（新增组） | 不为空，≤200字 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 优先级 | 分配优先级 | 输入 | 是 | 1（默认组）/ 空（新增组） | 正整数，数字越小优先级越高，同一方案内不可重复 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 评价规则 | 评分方式 | 输入 | 是 | 空 | 不为空，引用的评价规则必须已启用 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 考核组模式 | 按条件/按人员 | 输入 | 是 | 空 | 不为空 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 条件类型 | 部门/职位/职级/划分区域/人员类型/人员状态/手动选人 | 输入 | 按条件模式时必填 | 空 | 不为空 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 条件值 | 具体条件值 | 输入 | 按条件模式时必填 | 空 | 不为空，多选 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 排除人员 | 不纳入考核的人员 | 输入 | 否 | 空 | 仅可搜索选择条件范围下的人员 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |
| 指定人员 | 直接指定的考核成员 | 输入 | 按人员模式时必填 | 空 | 不为空，多选 | 新建、草稿、已启用、已停用重新编辑时均可编辑 |

#### 后端校验规则

| 校验项 | 规则 | 错误提示 |
|--------|------|---------|
| 至少一个考核组 | 考核组数量≥1 | "请至少配置一个考核组" |
| 考核组模式必填 | 每个考核组必须选择模式 | "请选择考核组模式" |
| 评价规则必填 | 每个考核组必须选择 | "请选择评价规则" |
| 评价规则可用性 | 引用的评价规则必须已启用 | "所引用的评价规则已停用，请重新选择" |
| 条件/人员必填 | 每个考核组至少包含一条规则 | "请至少添加一条筛选条件" |
| 排除人员范围 | 排除人员必须在条件匹配范围内 | "排除人员不在条件匹配范围内" |
| 优先级唯一性 | 同一方案内，考核组优先级不可重复 | "考核组优先级不可重复，请调整后保存" |
| 优先级格式 | 仅允许正整数 | "优先级请输入正整数" |

#### 数据处理规则
- **优先级分配逻辑**：当员工同时匹配多个考核组时，按优先级（数字最小）分配到第一个匹配的组
- **条件匹配逻辑**：多条条件之间的关系为"且"（AND），即员工必须同时满足所有条件才纳入该考核组
- **排除优先**：排除人员优先于条件匹配，即使满足条件也会被排除
- **未命中考核组处理**：员工仅在命中某个考核组规则时，才纳入本方案考核范围。若员工未命中任何考核组，则不进入该方案的考核范围。此规则不阻断方案启用
- **建议前端能力**：提供考核组匹配人数预览功能，提示可能存在未覆盖人员，但不作为保存阻断项
- **数据来源**：评价规则下拉数据来自 V1 评价规则表（perf_eval_rule），仅展示已启用的记录
- **数据来源**：部门/职位/职级条件值来自组织架构和岗位数据

#### 异常情况及处理规则

| 异常场景 | 用户可见提示 | 系统处理方式 |
|---------|-------------|-------------|
| 评价规则被停用 | "所引用的评价规则已停用，请重新选择" | 阻断保存，标红评价规则字段 |
| 条件数据加载失败 | "条件数据加载失败，请刷新重试" | 条件值下拉显示空 |
| 考核组优先级重复 | "考核组优先级不可重复，请调整后保存" | 阻断保存，标红优先级字段 |

### 4.11 方案与活动关系

#### 功能目标
定义方案配置如何传递到活动实例，确保配置隔离和版本管理。

#### 前端交互逻辑
- 方案详情页不展示关联活动详情；方案列表页统计展示关联活动数量

#### 数据处理规则

**快照规则**（后端业务规则）：
1. 活动创建时复制当时的方案全部配置，形成不可变的活动快照
2. 活动严格按创建时复制的方案快照执行，不可修改从方案继承的任何配置（考核组、环节、流程、等级、强制分布、通知等）
3. 基本信息（名称、周期类型、等级规则等）copy 后不可修改
4. 执行时只看活动快照配置，不回溯方案配置
5. 如需调整配置，应修改方案并创建新活动，或复制方案形成新方案后再创建活动

**快照隔离规则**：
1. 方案修改、停用、删除均不影响已创建的活动
2. 历史活动查看时展示活动创建时使用的方案快照内容
3. 已删除方案的历史活动仍保留对该方案的快照展示

**方案停用规则**：
1. 方案停用后，已创建的活动不受影响，继续按活动快照配置执行
2. 停用后的方案不可发起新活动，但可查看、复制、重新启用
3. 复制停用方案时，生成新方案（编号重新生成，版本重置为1，状态为草稿）

**方案版本规则**：
1. 每次保存方案配置，版本号自动+1
2. 版本历史自动记录，可查看历史版本的配置快照
3. 已启用的方案编辑保存后，版本号+1，仅影响后续新创建的活动

## 六、评分计算规则（V3/V4 执行参考）

> 本章节描述评分计算规则，由 V3 表单填写时和 V4 活动执行时使用。V2 方案配置中通过引用等级规则、评价规则、计算规则等方式预设这些规则。

### 6.1 定量指标得分

```
定量指标得分 = (实际值 - 保底值) / (挑战值 - 保底值) × 指标满分
```

- 结果四舍五入保留 1 位小数
- 实际值超出挑战值时，得分 = 指标满分
- 实际值低于保底值时，得分 = 0

### 6.2 定性指标得分

```
定性指标得分 = 评价规则中对应等级的分值
```

- 由评价规则（V1.7）配置每个等级的分值
- 评分人选择等级后，系统自动查表获取分值

### 6.3 单表单总分

```
单表单总分 = Σ(各指标得分 × 指标权重 / 100) + 加分 - 减分
```

- 指标权重以百分比存储（如 25 表示 25%）
- 加分/减分为手动输入值，范围 0~20 分
- 单表单总分上限 120 分，下限 0 分

### 6.4 最终得分（按节点权重加权）

```
最终得分 = Σ(各节点表单总分 × 节点权重) / Σ(节点权重)
```

- 节点权重以百分比存储（如 40+60=100）
- 未配置节点权重时，最终得分 = 最后一个填写节点的表单总分

### 6.5 等级映射

```
等级 = grade where grade.min ≤ 总分 < grade.max
```

- 总分按等级规则的分数区间映射到等级
- 等级仅用于展示和强制分布，不参与计算

## 七、结果确认与绩效申诉（V4 执行参考）

> 本章节描述结果确认流程和绩效申诉处理方式，由 V4 活动执行时使用。V2 本版本仅配置结果确认环节及流程模板；员工不同意、BP 知悉、线下处理属于 V4 执行层定义，不在本版本方案管理中落地。

### 结果确认流程

1. 审定完成后，员工收到确认待办
2. 确认表单展示：绩效考核结果（只读）+ 审定结果（只读）+ 面谈结果（只读，若面谈环节已开启）
3. 员工选择"同意"→ 流程结束，结果锁定
4. 员工选择"不同意"→ 填写不同意意见 → 进入 BP 知悉节点
5. BP 知悉后 → 流程结束，结果锁定，线下处理

### 确认超时规则

- 员工确认环节超时未操作时，系统自动确认为"同意"

### 绩效申诉说明

- V2 不做独立的绩效申诉模块
- 员工对结果有异议时，通过结果确认环节的"不同意"入口反馈
- BP 知悉后线下沟通处理

|------|--------|------|-----------|
| `draft` | 草稿 | 新建或编辑中，配置未完成 | 否 |
| `enabled` | 已启用 | 配置完整，可用于发起活动 | 是 |
| `disabled` | 已停用 | 已启用后被停用，不可发起新活动 | 否 |

**状态进入规则**：
- 用户点击"新增"进入向导 → 后端创建草稿记录
- 切换步骤暂存、校验失败、保存草稿 → 状态保持草稿
- 点击"保存"且全量校验通过 → 状态按"是否启用"字段变为已启用或已停用

**状态流转规则**：

| 来源状态 | 触发动作 | 目标状态 | 前置条件 | 操作角色 | 不允许操作 |
|---------|---------|---------|---------|---------|-----------|
| 草稿 | 保存（启用=是） | 已启用 | 全量校验通过 | SSC | — |
| 草稿 | 保存（启用=否） | 已停用 | 全量校验通过 | SSC | — |
| 草稿 | 删除 | —（逻辑删除） | 无前置校验 | SSC | — |
| 已启用 | 停用 | 已停用 | 关联进行中活动数=0 | SSC | 有关联进行中活动时阻断 |
| 已启用 | 编辑保存（启用=是） | 已启用 | 全量校验通过，版本号+1 | SSC | — |
| 已启用 | 编辑保存（启用=否） | 已停用 | 全量校验通过 + 关联进行中活动数=0，版本号+1 | SSC | 有关联进行中活动时阻断，保持已启用 |
| 已停用 | 启用 | 已启用 | 完整校验（基本信息+考核组+环节+引用配置可用性）通过 | SSC | 校验失败则保持已停用 |
| 已停用 | 删除 | —（逻辑删除） | 已创建的活动不受影响 | SSC | — |
| 已停用 | 复制 | 草稿（新方案） | 生成新编号，版本重置为1 | SSC | — |

**异常处理**：
- 保存校验失败：返回错误汇总，保持草稿状态
- 重新启用校验失败：返回错误汇总，保持已停用状态
- 停用时存在进行中活动：阻断操作，提示"当前方案存在进行中的绩效活动，不可停用"
- 已启用编辑保存选择停用但存在进行中活动：阻断停用，保持已启用状态，提示"当前方案存在进行中的绩效活动，不可停用，请将"是否启用"改回"启用"或等待活动完成"
- 并发操作（同时启用/停用）：乐观锁机制，后操作者收到冲突提示

## 附录

### A. 术语表

| 术语 | 定义 |
|------|------|
| SSC | Shared Service Center，共享服务中心，负责在系统中统一维护绩效方案配置 |
| BP | 业务伙伴角色，负责部分绩效活动执行数据处理；不参与方案管理 |
| COE | Center of Expertise，专家中心，业务政策角色，不参与本系统方案管理操作 |
| PBC | Personal Business Commitment，个人业务承诺，绩效目标管理方式 |
| 考核组 | 方案中定义的考核人员分组，按条件或按人员划分 |
| 考核环节 | 绩效考核的各个阶段：目标制定、绩效考核、结果审定、结果确认、绩效面谈 |
| 强制分布 | 要求各等级人数比例符合预设要求的管控机制 |
| 流程模板 | 流程引擎中定义的流程模型，定义审批节点和流转规则 |
| 动态考核 | 每次环节启动时实时查询当前汇报关系（vs 开启时锁定） |
| 向下公开 | 控制员工是否可在工作台查看该方案关联的绩效结果 |

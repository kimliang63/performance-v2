## 二、产品背景与目标

### 2.1 V4 在整体绩效系统中的位置

系统采用三层配置架构：

```
V1 基础配置层（COE 维护）
  绩效周期 / 等级规则 / 计算规则 / 衡量标准 / 指标库 / 评价规则 / 强制分布规则
      ↓ 引用（只读）
V2 方案配置层（COE 维护）
  绩效方案 / 考核组 / 考核环节 / 流程模板 / 表单模板 / 强制分布参数
      ↓ 复制全部配置生成不可变快照 + 绑定具体周期
V4 活动执行层（HRBP 操作）
  考核活动 / 参与者 / 环节状态 / 操作日志
```

V4 是绩效系统的执行引擎：基于 V2 方案模板实例化为活动，驱动 V3 表单的填写与审批流程，管控考核进度和异常。

### 2.2 V4 解决的核心问题

1. 将方案配置从"模板"变为"实例"：一次方案可跨多个周期复用
2. 提供活动全生命周期管控：创建→执行→监控→干预→归档
3. 支持 HRBP 按权限对活动进行干预操作（开启/关闭/暂停/终止/催办/转交）
4. 提供活动进度监控和异常预警能力
5. 支持批量数据导入导出，降低线下操作成本

### 2.3 主要用户与使用场景

| 角色 | 使用场景 |
|------|----------|
| HRBP | 发起活动、管控环节、导入导出数据、监控进度、处理异常、催办/转交 |
| COE | 查看活动列表和操作日志（只读），了解方案执行情况 |
| SSC | 结果确认不同意后续处理（按流程配置）；查看活动列表和操作日志（只读） |
| 直线上级 | 作为流程处理人参与表单填写和审批（V3 范围）；结果确认不同意后续处理（按流程配置） |
| 双线上级 | 作为流程处理人参与表单填写和审批（V3 范围） |
| 员工/被考核人 | 参与表单填写和确认（V3 范围） |

### 2.4 与 V1/V2/V3/V5/V6 的边界

| 版本 | 与 V4 的关系 |
|------|-------------|
| V1 | V4 活动使用 V1 基础配置的快照（等级规则、计算规则、评价规则、强制分布规则），活动进行中不受 V1 修改影响 |
| V2 | V4 发起活动时选择已启用方案，复制方案全部配置生成不可变快照；V4 活动级可调整通知配置和未开始环节的开启方式、时间范围 |
| V3 | V3 定义表单字段和流程节点，V4 管控活动和环节状态；V4 环节开启/关闭触发 V3 流程实例的创建和终止 |
| V5 | 员工工作台展示 V4 活动数据（待办、进度、结果），不在 V4 展开 |
| V6 | 智能辅助能力属于后续版本，不在 V4 范围内 |

## 四、通用业务规则

### 4.1 活动与方案关系

- 活动发起时必须选择已启用且未逻辑删除的方案
- 活动发起时从方案复制全部配置生成不可变快照，活动运行期间始终读取快照配置
- 活动发起时选择具体绩效周期，周期信息同步快照到活动
- 活动进行中不受方案后续修改影响
- 活动进行中不受基础配置修改影响

### 4.2 活动级配置覆盖

活动可调整的配置项（从方案快照继承）：

| 可调整配置 | 说明 |
|-----------|------|
| 通知配置 | 通知类型×渠道 |
| 未开始环节的开启方式 | 定时/手动，可按活动调整 |
| 未开始环节的时间范围 | 活动级设置开始/结束时间 |

活动不可调整的配置（严格按方案快照执行）：

| 不可调整配置 | 说明 |
|-------------|------|
| 考核组规则 | 人员范围由方案快照决定 |
| 流程模板 | 由方案快照决定 |
| 表单模板 | 由方案快照决定 |
| 等级规则 | 由方案快照决定 |
| 计算规则 | 由方案快照决定 |
| 强制分布参数 | 分布比例、控制模式（强控/弱控），由方案快照决定 |
| 节点权重 | 绩效考核环节的节点评分权重，由方案快照决定 |

> 说明：强制分布参数和节点权重在 V2 方案配置时确定，活动发起后按快照执行。如需在活动级调整，需通过方案修改后重新发起活动。活动级是否需要微调强制分布参数或节点权重，待业务确认。

### 4.3 活动状态规则

| 状态 | 描述 | 可执行操作 |
|------|------|------------|
| 草稿 | 活动已创建未启动 | 编辑、启用、删除 |
| 进行中 | 活动正在执行 | 环节管控、导入导出、监控、干预 |
| 已挂起 | 活动被暂停，所有环节冻结 | 恢复、终止 |
| 已完成 | 所有环节完成 | 查看、导出、归档 |
| 已终止 | 手动终止 | 查看、导出 |

### 4.4 环节状态规则

每个参与者在每个环节有独立的状态：

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| 未开始 | 环节未开启，或参与者未进入该环节 | 环节未开启 |
| 处理中 | 环节已开启，待办已分配，等待处理 | 环节开启后 |
| 已完成 | 处理人提交表单，流程结束 | 处理人提交 |
| 已跳过 | 环节配置为跳过，或审批人为空自动跳过 | 配置跳过或审批人为空 |
| 已超时 | 超过配置时间未处理 | 超过截止时间 |
| 已暂停 | 被 HRBP 手动暂停 | HRBP 执行暂停操作 |

### 4.5 人员状态规则

| 状态 | 含义 | 说明 |
|------|------|------|
| 正常参与 | 参与者正在流程中 | 默认状态 |
| 已完成 | 所有环节已完成 | 所有环节均已处理 |
| 已终止 | 流程被强制终止 | HRBP 执行终止操作 |
| 已移除 | 从活动中移除 | HRBP 执行移除操作 |
| 已暂停 | 被暂停，等待恢复 | HRBP 执行暂停操作 |

### 4.6 操作原因填写规则

以下操作必须填写操作原因：

| 操作 | 是否必填 | 说明 |
|------|---------|------|
| 终止活动 | 必填 | 终止原因 |
| 终止单人流程 | 必填 | 终止原因 |
| 重开环节 | 必填 | 重开原因 |
| 退回流程 | 必填 | 退回原因 |
| 转交流程 | 必填 | 转交原因 |
| 暂停环节/人员 | 选填 | 暂停原因 |
| 移除人员 | 必填 | 移除原因 |

### 4.7 操作留痕规则

所有关键操作必须记录：

| 记录内容 | 说明 |
|---------|------|
| 操作人 | 执行操作的用户 |
| 操作时间 | 操作执行时间 |
| 操作类型 | 具体操作类型 |
| 操作对象 | 被操作的活动/环节/人员 |
| 操作前状态 | 操作前的状态 |
| 操作后状态 | 操作后的状态 |
| 操作原因 | 填写的操作原因（必填时） |

### 4.8 批量操作规则

- 批量操作按勾选项逐项处理
- 部分失败时，成功的正常执行，失败的返回错误明细
- 批量操作不做原子性保证
- 批量催办、批量转交、批量暂停、批量恢复均按此规则

### 4.9 导入导出通用规则

- 导入文件格式：Excel (.xlsx) 或 CSV
- 文件大小限制：单文件 ≤ 10MB
- 数据行数限制：单次 ≤ 5000 行
- 编码要求：UTF-8
- 导入采用逐行校验，部分成功部分失败模式
- 导入结果需展示成功/失败条数和失败原因

### 5.2 发起活动

#### 业务场景

HRBP 基于已启用的绩效方案，选择具体绩效周期，创建一次考核活动。活动创建时从方案复制全部配置生成不可变快照。

#### 用户角色

HRBP

#### 用户操作流程

1. HRBP 在活动列表页点击"发起活动"
2. 弹出发起活动弹窗
3. 选择关联方案（仅展示已启用且未逻辑删除的方案）
4. 选择考核周期（与方案周期类型匹配的可用周期）
5. 填写活动名称
6. 选择考核关系（开启时锁定/动态获取）
7. 填写活动说明（选填）
8. 确认发起

#### 弹窗字段规则

| 字段 | 是否必填 | 默认值 | 校验规则 |
|------|---------|--------|---------|
| 活动名称 | 必填 | 空 | 不为空，≤200 字符，同语种唯一 |
| 关联方案 | 必填 | 空 | 仅展示已启用且未逻辑删除的方案 |
| 考核周期 | 必填 | 空 | 仅展示与方案周期类型匹配的可用周期 |
| 考核关系 | 必填 | 开启时锁定 | 开启时锁定/动态获取 |
| 活动说明 | 选填 | 空 | ≤2000 字符 |

#### 发起后系统处理

1. 系统校验必填项
2. 从方案复制全部配置生成不可变快照，活动运行期间始终读取快照配置
3. 从绩效周期复制周期信息生成不可变快照

### 5.5 活动删除/取消/终止

#### 活动删除

**适用状态：** 草稿状态

**操作流程：**
1. HRBP 点击"删除"
2. 弹出二次确认："确定要删除「活动名称」吗？此操作不可撤销。"
3. 确认后逻辑删除，活动从列表消失

**校验：** 草稿状态无前置校验。

#### 活动终止

**适用状态：** 进行中

**操作流程：**
1. HRBP 点击"终止活动"
2. 弹出终止确认弹窗，要求填写终止原因（必填）
3. 确认后活动状态变为"已终止"
4. 所有处理中的环节冻结，所有未完成的流程实例终止
5. 系统记录操作日志

**终止后影响：**
- 活动状态变为"已终止"
- 所有处理中的环节状态变为"未开始"
- 所有未完成的流程实例终止
- 已完成的数据保留，不可再修改
- 终止后不可恢复

#### 活动归档

**适用状态：** 已完成

**操作流程：**
1. HRBP 点击"归档"
2. 系统检查是否有未完成的环节
   - 有未完成环节：提示"以下环节未完成：[环节名] [X]人未完成，请先到对应环节手动结束"
   - 无未完成环节：展示归档确认弹窗（活动名称、人数、完成率）
3. 确认后活动状态变为"已归档"，数据锁定

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-4.5.1 | 草稿状态可删除，需二次确认 |
| AC-4.5.2 | 进行中状态可终止，需填写原因 |
| AC-4.5.3 | 终止后所有处理中环节冻结 |
| AC-4.5.4 | 已完成状态可归档，需检查未完成环节 |
| AC-4.5.5 | 归档后数据锁定不可修改 |

### 5.6 人员选择

#### 业务场景

HRBP 在活动创建后或活动进行中，管理活动的参与人员范围。

#### 人员匹配规则

- 活动创建时，按方案快照中配置的考核组规则自动匹配参与者
- 每个员工按考核组优先级分配到第一个命中的考核组
- 未命中任何考核组的员工不参与考核
- 同一员工只能参与一次活动（去重）

#### 人员添加方式

| 方式 | 说明 |
|------|------|
| 按考核组规则更新 | 重新按方案快照的考核组规则匹配，新增符合条件的人员 |
| 手动添加 | HRBP 手动搜索并添加指定人员 |
| 导入添加 | 通过 Excel 文件批量导入人员 |

#### 人员移除

**适用条件：**
- 被移除人员在所有环节中均为"未开始"状态
- 已进入任何环节流程的人员不可移除

**操作流程：**
1. HRBP 选择要移除的人员
2. 弹出移除确认，要求填写移除原因（必填）
3. 确认后从活动参与者列表中移除

#### 活动中新增/移除人员的业务限制

| 场景 | 是否允许 | 说明 |
|------|---------|------|
| 草稿状态添加人员 | ✅ 允许 | 通过手动添加或按规则更新 |
| 草稿状态移除人员 | ✅ 允许 | 无限制 |
| 进行中添加人员 | ✅ 允许 | 新增人员从第一个环节开始参与 |
| 进行中移除人员 | ⚠️ 受限 | 仅可移除所有环节均为"未开始"的人员 |
| 已完成/已终止添加人员 | ❌ 不允许 | — |
| 已完成/已终止移除人员 | ❌ 不允许 | — |

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-4.6.1 | 活动创建时自动按考核组规则匹配人员 |
| AC-4.6.2 | 按规则更新后，新增人员从第一个环节开始 |
| AC-4.6.3 | 手动添加时，搜索范围为组织架构内所有员工 |
| AC-4.6.4 | 移除人员时，已进入流程的人员不可移除 |
| AC-4.6.5 | 移除操作需填写原因并记录日志 |

### 6.1 开启环节

#### 业务场景

HRBP 手动开启活动中的某个考核环节，使该环节的参与者开始处理。

#### 谁可以操作

HRBP

#### 可操作状态

- 活动状态为"进行中"
- 要开启的环节状态为"未开始"
- 该环节的前置环节已完成（如有依赖关系）

#### 环节依赖关系

| 环节 | 前置依赖 |
|------|---------|
| 目标制定 | 无（活动启用后可直接开启） |
| 绩效考核 | 目标制定已完成 |
| 结果审定 | 绩效考核已完成 |
| 结果确认 | 结果审定已完成 |
| 绩效面谈 | 无（可与目标制定并行开启） |

#### 是否需要填写原因

否

#### 是否需要二次确认

是。弹出确认："确定开启「环节名称」环节吗？开启后将为 X 名参与者分配待办任务。"

#### 操作后影响哪些人

该环节的所有参与者

#### 操作后流程变化

1. 环节状态变为"处理中"
2. 为该环节的参与者创建流程实例
3. 按流程模板分配待办任务
4. 相关处理人收到待办通知

#### 通知规则

- 开启环节后，系统自动向该环节的待办处理人发送待办通知
- 通知渠道：站内信 + 飞书（按活动通知配置）

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-6.1.1 | 环节状态为"未开始"时可开启 |
| AC-6.1.2 | 前置环节未完成时不可开启后续环节 |
| AC-6.1.3 | 开启后为参与者创建流程实例并分配待办 |
| AC-6.1.4 | 开启后处理人收到通知 |
| AC-6.1.5 | 操作记录到操作日志 |

HRBP

必填

   - **已提交且已审批通过的数据：保留**，不重新执行
   - **已提交但未审批的数据：保留**，继续流转
   - **未完成的参与者：重新进入流程**，从该环节的第一个节点开始
   - **已跳过的参与者：重新进入流程**
3. 重新为未完成的参与者分配待办任务

#### 通知规则

- 重开后，向需要重新处理的参与者发送待办通知

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-6.6.1 | 已关闭/已终止/已跳过的环节可重开 |
| AC-6.6.2 | 重开需填写原因 |
| AC-6.6.3 | 已完成的参与者数据保留不重新执行 |
| AC-6.6.4 | 未完成的参与者重新进入流程 |
| AC-6.6.5 | 重开后参与者收到待办通知 |

### 6.7 强制终止

#### 业务场景

HRBP 强制终止某个参与者的流程，不再接受其提交。

#### 谁可以操作

HRBP

#### 可操作状态

- 活动状态为"进行中"
- 被终止的参与者状态为"处理中"（在某个环节的流程中）

#### 是否需要填写原因

必填

#### 是否需要二次确认

是。弹出确认："确定终止 [员工姓名] 的流程吗？终止后将使用方案配置的默认结果。"

#### 操作后影响

1. 该参与者的当前流程实例终止
2. 该参与者在当前环节的状态变为"已终止"
3. 使用方案配置的默认结果（默认等级/默认评价结果）
4. 该参与者可被重开（如 HRBP 需要恢复其流程）
5. 不影响其他参与者

#### 活动终止 vs 环节终止 vs 单人流程终止 的区别

| 终止类型 | 影响范围 | 是否可恢复 | 默认结果来源 |
|---------|---------|-----------|------------|
| 活动终止 | 整个活动，所有参与者 | 不可恢复 | 无需默认结果 |
| 环节终止 | 某环节的所有参与者 | 可重开环节 | 方案配置的默认结果 |
| 单人流程终止 | 单个参与者 | 可重开该人员 | 方案配置的默认结果 |

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-6.7.1 | 可终止单个参与者的流程 |
| AC-6.7.2 | 终止后使用方案配置的默认结果 |
| AC-6.7.3 | 终止不影响其他参与者 |
| AC-6.7.4 | 终止原因必填 |

### 6.13 结果确认不同意后续处理

#### 业务场景

员工在结果确认环节选择"不同意"后，流程进入后续处理分支。HRBP/SSC/直线上级作为后续处理人，查看员工异议并做记录、跟进和关闭处理。

#### 谁可以操作

按流程模板配置的后续处理节点角色，当前业务口径为：HRBP、SSC、直线上级。

#### 触发条件

员工在结果确认环节选择"不同意"并填写不同意原因。

#### 后续处理人可见内容

| 内容 | 是否可见 | 说明 |
|------|---------|------|
| 员工不同意原因 | ✅ 可见 | 员工填写的异议说明 |
| 绩效考核结果 | ✅ 可见 | 考核总分、各指标得分（只读） |
| 审定结果 | ✅ 可见 | 审定等级（只读） |
| 面谈记录 | ✅ 可见 | 面谈内容（只读，若已启用面谈环节） |
| 相关操作日志 | ✅ 可见 | 该员工在各环节的操作记录 |

#### 后续处理操作

后续处理只做记录、跟进和关闭，不改变员工确认的终态结果：

| 操作 | 说明 |
|------|------|
| 查看异议详情 | 查看员工不同意原因和相关考核数据 |
| 填写处理意见 | 记录后续处理意见和跟进措施 |
| 关闭事项 | 确认处理完成，关闭该待处理事项 |

#### 关键业务规则

- 后续处理**不让员工重新确认**，结果确认状态为终态不可逆
- 后续处理**不回退已终态的结果确认**，不触发流程重来
- 后续处理完成后，该事项关闭并记录操作日志
- 后续处理只做记录和跟进，争议线下沟通处理

#### 字段规则

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 处理意见 | 必填 | 后续处理人的处理说明 |
| 关闭原因 | 必填 | 关闭该事项的原因 |

#### 异常提醒

- 员工选择"不同意"后，系统在活动监控的"待处理事项"中生成一条待处理记录
- 待处理事项展示：员工姓名、工号、部门、不同意原因、生成时间
- 如后续处理超时未处理，系统催办后续处理人

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-6.13.1 | 员工选择"不同意"后，待处理事项中生成后续处理记录 |
| AC-6.13.2 | 后续处理人可查看员工不同意原因和相关考核数据 |
| AC-6.13.3 | 后续处理人填写处理意见后可关闭事项 |
| AC-6.13.4 | 关闭后记录操作日志 |
| AC-6.13.5 | 后续处理不改变员工确认的终态结果 |
| AC-6.13.6 | 后续处理不触发流程重来 |

### 8.8 提交率预警

#### 业务场景

### 8.9 进度滞后预警

#### 业务场景

系统监控活动整体进度，低于预期时预警。

#### 统计口径

```
活动整体进度 = 已完成所有环节数 / (参与人数 × 已开启环节数) × 100%
```

#### 预警阈值

| 预警级别 | 条件 | 提醒对象 |
|---------|------|---------|
| 黄色预警 | 整体进度 < 预期进度 20% | HRBP |
| 红色预警 | 整体进度 < 预期进度 40% | HRBP + HRBP 主管 |

#### 预期进度计算

系统根据活动时间和当前日期，计算预期应完成的进度百分比。预期进度 = (当前日期 - 活动开始日期) / (活动结束日期 - 活动开始日期) × 100%

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-8.9.1 | 进度计算正确 |
| AC-8.9.2 | 低于预期时触发预警 |
| AC-8.9.3 | 预警通知发送到正确对象 |

操作人

#### 提醒时机

操作执行时立即提示

#### 页面提示文案

- "当前活动状态不允许此操作"

#### 是否阻断操作

是

#### 验收标准

|------|---------|

### 9.9 流程处理人缺失

#### 触发条件

- 流程节点的处理人为空（如双线上级为空）
- 流程模板配置的处理人规则无法匹配到有效处理人

#### 提醒对象

HRBP

#### 提醒时机

环节开启时检测，发现缺失立即提醒

#### 页面提示文案

- "[环节名称] 的 [节点名称] 缺少处理人，该节点将自动跳过"

#### 后续处理动作

- 系统按配置规则自动跳过空处理人节点
- HRBP 可转交或调整处理人

#### 是否阻断操作

否（自动跳过），仅提醒

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-9.9.1 | 处理人缺失时提醒 HRBP |
| AC-9.9.2 | 空处理人节点自动跳过 |

### 9.10 配置缺失或不可用

#### 触发条件

- 活动快照中的配置项被引用但不可用（如流程模板被停用）
- 活动级覆盖配置与快照配置冲突

#### 提醒对象

HRBP

#### 提醒时机

活动执行中检测到配置异常时

#### 页面提示文案

- "[配置项] [配置名称] 当前不可用，请联系管理员"

#### 后续处理动作

- HRBP 联系 SSC 修复配置
- 活动继续使用快照配置（不受方案修改影响）

#### 是否阻断操作

视情况而定（严重配置缺失可能阻断环节开启）

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-9.10.1 | 配置异常时提醒 HRBP |
| AC-9.10.2 | 活动使用快照配置不受影响 |

## 十一、状态流转

---

### 11.1 活动状态

```
              ┌──────────────┐
              │    草稿       │
              └──────┬───────┘
                     │ 启用
                     ▼
              ┌──────────────┐
              │    进行中     │◄─── 恢复
              └──────┬───────┘
            ┌────────┼────────┐
            ▼        ▼        ▼
      ┌──────────┐ ┌──────────┐ ┌──────────┐
      │  已完成   │ │  已终止   │ │  已挂起   │
      └────┬─────┘ └──────────┘ └──────────┘
           │ 归档
           ▼
      ┌──────────┐
      │  已归档   │
      └──────────┘
```

| 来源状态 | 触发动作 | 目标状态 | 前置条件 |
|---------|---------|---------|---------|
| 草稿 | 启用 | 进行中 | 必填项完整 |
| 草稿 | 删除 | —（逻辑删除） | — |
| 进行中 | 完成 | 已完成 | 所有环节完成 |
| 进行中 | 终止 | 已终止 | 填写终止原因 |
| 进行中 | 挂起 | 已挂起 | 填写挂起原因 |
| 已挂起 | 恢复 | 进行中 | — |
| 已完成 | 归档 | 已归档 | 所有环节已完成 |

---

### 11.2 环节状态

```
              ┌──────────────┐
              │   未开始      │
              └──────┬───────┘
                     │ 开启
                     ▼
              ┌──────────────┐
              │   处理中      │◄─── 恢复
              └──────┬───────┘
            ┌────────┼────────┐
            ▼        ▼        ▼
      ┌──────────┐ ┌──────────┐ ┌──────────┐
      │  已完成   │ │  已暂停   │ │  已终止   │
      └──────────┘ └──────────┘ └──────────┘
            ▲
            │ 重开
      ┌──────────┐
      │  已跳过   │
      └──────────┘
```

| 来源状态 | 触发动作 | 目标状态 | 说明 |
|---------|---------|---------|------|
| 未开始 | 开启 | 处理中 | 前置环节已完成 |
| 处理中 | 关闭 | 已完成 | 手动结束 |
| 处理中 | 暂停 | 已暂停 | 暂停后不可提交 |
| 处理中 | 终止 | 已终止 | 终止后使用默认结果 |
| 已暂停 | 恢复 | 处理中 | 恢复后继续 |
| 已终止 | 重开 | 处理中 | 已完成数据保留 |
| 已完成 | 重开 | 处理中 | 未完成人员重新进入 |

## 十二、与其他版本的边界

| 版本 | 职责边界 | 与 V4 的交互 |
|------|---------|-------------|
| V1 基础配置 | 维护周期、等级规则、计算规则等 | V4 活动使用 V1 配置的快照，活动进行中不受 V1 修改影响 |

| V5 员工工作台 | 员工端操作界面 | V5 引用 V4 活动数据展示待办、进度、结果 |
| V6 智能辅助能力 | 智能辅助能力 | 不在 V4 范围内 |

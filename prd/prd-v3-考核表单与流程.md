## 三、通用业务规则

本章定义所有表单和流程共用的业务规则。各表单章节引用本章规则，不再重复。

### 3.1 正式提交规则

- 系统不提供正式草稿能力，页面保存仅作为本地临时暂存
- 所有正式业务操作（填写提交、审批通过、审批驳回、确认同意/不同意）统一通过"提交"完成
- 提交需完成校验、计算、记录结果并推进流程；任一环节失败则本次提交不生效

### 3.2 审批整表处理规则

审批是对整张表单的审批，不存在"单条指标审批"或"单条目标审批"。

每次提交、每次审批，都是一条独立的业务记录。但两种记录性质不同：

| 记录类型 | 性质 | 说明 |
|---------|------|------|
| 员工提交记录 | 业务数据 | 员工填写的表单内容，是业务事实的载体 |
| 审批动作记录 | 动作日志 | 审批人对该次员工提交的审批动作（通过/驳回），不承载业务数据 |

**关键规则：**
- 审批记录不改变业务数据内容
- 查看当前生效数据时，取"最近一次审批通过所引用的员工提交记录"
- 被驳回的员工提交记录标记为失效，员工需重新提交新版本
- 所有历史记录可追溯

### 3.3 驳回与重提规则

**逐级驳回原则：**
- 所有审批节点都可以驳回
- 流程的首节点（发起/填写/评级节点）不可驳回，因为不存在"上一个节点"
- 当前审批节点驳回时，退回到上一个可处理节点

**各环节驳回规则：**

| 环节 | 驳回方 | 驳回后退到 | 重提方式 |
|------|-------|-----------|---------|
| 目标制定 | 双线上级 | 退回员工填写节点 | 员工修改后重新提交 |
| 目标制定 | 直线上级 | 由流程模板配置决定（可退双线上级或员工） | 对应节点处理人重新处理 |
| 目标批量导入 | 直线上级 | 退回 HRBP 节点 | HRBP 重新导入并提交 |
| 绩效考核 | 双线上级 | 退回员工自评节点 | 员工修改后重新提交 |
| 绩效考核 | 直线上级 | 退回双线上级评分节点 | 双线上级重新处理 |
| 考核结果批量导入 | 直线上级 | 退回 HRBP 节点 | HRBP 重新导入并提交 |
| 结果审定 | 双线上级/逐级上级/HRBP | 退回上一个可处理节点 + 强制分布回滚 | 对应节点处理人重新处理 |
| 结果确认 | 不支持驳回 | — | — |
| 绩效面谈 | 直线上级 | 退回员工填写节点 | 员工修改后重新提交 |

**驳回后的记录处理：**
- 被驳回链路上的未生效员工提交记录标记为失效
- 审批动作记录保留，用于追溯
- 员工重新提交时生成新的员工提交记录

### 3.4 评分计算规则

绩效考核环节的计算分为三层：

**第一层：指标得分（每条指标独立计算，员工自评提交或考核结果批量导入提交时执行）**

| 指标类型 | 计算方式 |
|---------|---------|
| 量化指标 | 根据指标绑定的计算规则，用员工自评或考核结果批量导入提交的完成值和衡量标准梯度值自动计算得分。如果指标缺少计算规则，系统拒绝提交并提示"指标计算规则未配置，请联系管理员" |
| 定性指标 | 员工自评或考核结果批量导入提交的评价结果等级，系统按评价规则中该等级对应的分值取分 |

**第二层：节点表单总分（员工自评提交或考核结果批量导入提交时计算）**
- 公式：各指标得分按权重加权求和，加上加分、减去减分
- 范围：0–120 分
- 双线上级/直线上级提交时不重新计算此分数

**第三层：最终考核得分（所有考核节点完成后计算）**
- 公式：各有效得分节点的表单总分按方案配置的节点权重加权求和
- 等级映射：最终得分落在某个等级的分数区间内，即映射为该等级
- 等级仅用于展示和强制分布统计，不参与计算

**上级节点的评分行为：**
- 上级不可修改员工填写的完成值、完成情况和评价结果
- 上级可填写：本节点的指标评价（文字，不参与得分计算）、总评语、初评等级（用于审定参考）
- 上级提交时不重新计算逐指标得分，仅记录本节点评价数据

### 3.5 强制分布规则

- 强制分布按绩效方案中配置的规则执行
- 每次审定提交后，系统重新计算各等级当前人数和占比
- **分母** = 考核组内所有员工数（含跳过审定和未审定员工）
- **分子** = 该等级的已审定有效员工数 + 跳过审定/强制结束的员工数（使用方案配置的默认等级）
- 仍在审定流程中、尚未产生有效等级的员工，只计入分母，不计入任何等级分子
- **强控模式**：超出限制时阻断提交
- **弱控模式**：超出限制时允许提交，但返回警告提示
- 并发审定同一员工时，后提交者需等待前一提交完成，防止分布计数不一致

### 3.6 跳过与强制结束规则

- 跳过审定或强制结束时，使用绩效方案配置的默认等级
- 如果只是跳过某一个审定节点（而非全流程跳过），后续审定节点仍可调整等级
- 跳过/强制结束的员工仍纳入强制分布分母，其默认等级计入对应等级分子
- 后续节点调整后，以最新有效审定等级重新参与统计

### 3.7 结果确认规则

- 结果确认采用状态驱动模型：每位员工在确认表中只有一条记录
- 员工操作（同意/不同意）直接更新该记录状态
- **终态不可逆**：一旦确认完成，无法撤销或修改
- **不支持驳回**：系统不存在"驳回员工重新确认"的操作
- 不同意后，流程进入结果确认后续处理分支，后续处理节点由流程模板配置决定
- 当前默认业务配置为：HRBP / SSC / 直线上级处理
- 超时未操作时，系统自动以"同意"身份完成确认

### 3.8 权限与可见性规则

| 规则 | 说明 |
|------|------|
| 员工数据隔离 | 员工只能查看和操作自己的数据 |
| 上级可见范围 | 上级可查看直线下级和双线下级的数据 |
| HRBP 可见范围 | HRBP 可查看所辖部门的数据 |
| SSC 可见范围 | SSC 可查看所有数据 |
| 审批数据范围 | 审批人只能查看被审批的员工提交内容 |
| 强制分布数据 | 仅审定人和 HRBP 可查看实时强制分布统计 |
| 历史记录 | 各角色可查看权限范围内的历史提交记录 |
| 操作留痕 | 所有提交、审批、确认操作必须记录操作人、操作时间、操作类型和操作结果 |

### 3.9 活动快照规则

- 活动创建时从绩效方案复制完整配置快照
- 活动运行期间始终读取快照配置，不受后续方案修改影响
- 方案被禁用或删除不影响已创建的活动

### 3.10 通用批量导入规则

本节定义所有批量导入场景共用的规则。目标批量导入、考核结果批量导入均引用本节。

#### 文件格式与限制

| 指标 | 要求 |
|------|------|
| 支持格式 | `.xlsx`、`.csv` |
| 单文件大小 | ≤ 10MB |
| 单次导入行数 | ≤ 5000 行 |
| 编码 | UTF-8 |

#### 导入流程

每次批量导入必须经历以下步骤，按顺序执行：

1. **上传**：操作人选择本地文件，系统接收并解析文件内容
2. **解析**：系统识别文件结构，按列名映射字段
3. **校验**：逐行校验字段内容（必填、格式、范围、引用有效性），标记成功行和失败行
4. **预览**：展示导入结果预览，包含成功条数、失败条数、失败行的具体原因
5. **确认提交**：操作人确认后，系统仅处理成功行，失败行不进入业务流程
6. **结果反馈**：展示最终导入结果（成功 N 人、失败 M 人），支持查看失败明细

#### 校验原则

- 导入范围必须与当前活动的考核组员工范围一致，不在范围内的员工行标记为失败
- 校验按行独立执行，某一行失败不影响其他校验成功行提交
- 失败行不进入业务流程，成功行进入后续审批流程
- 失败行必须展示到行级：行号、员工姓名/工号、失败字段、失败原因

#### 模板导出规则

- 模板必须按当前活动的考核组、表单模板、员工范围动态生成
- 模板字段与导入校验字段保持一致
- 不导出不在当前处理范围内的员工数据
- 模板表头包含字段说明（如"权重：整数，范围 1–100，所有目标合计须等于 100%"）

#### 操作留痕

每次导入操作必须记录以下信息：

| 记录项 | 说明 |
|--------|------|
| 操作人 | 执行导入的用户 |
| 操作时间 | 导入提交时间 |
| 文件名 | 上传的文件名 |
| 成功条数 | 校验通过并成功写入的行数 |
| 失败条数 | 校验未通过的行数 |
| 提交结果 | 导入后进入的流程节点（如"已进入直线上级审批节点"） |

### 4.1 PBC 目标制定表单

#### 业务场景

被考核人在绩效周期开始时填写个人绩效目标（PBC），经双线上级和直线上级审批后锁定，作为后续绩效考核的依据。目标数据单向流入绩效考核环节（只读引用）。

#### 使用角色

| 角色 | 操作 |
|------|------|
| 员工/被考核人 | 查看、填写、提交 |
| 双线上级 | 查看、审批（同意/驳回） |
| 直线上级 | 查看、审批（同意/驳回） |
| HRBP | 查看（审批通过后抄送） |

#### 用户进入入口

GHR → 绩效管理 → 考核活动 → 目标制定环节 → 员工目标表单

> 原型参考：`perf-activity.html`（活动详情页目标制定环节）、`perf-form-goal.html`（目标填写表单）。原型仅作交互参考，业务规则以本 PRD 为准。

#### 用户可见内容

**基本信息（只读）：** 被考核人姓名、工号、部门、职位、考核周期

**汇总信息（只读）：** 总权重、总目标数

**目标列表：** 每行包含目标名称、是否关联指标、关联指标、权重、描述、衡量标准（梯度值）、进度

#### 用户可填写字段

| 字段          | 是否必填 | 填写角色 | 说明                           |
| ----------- | ---- | ---- | ---------------------------- |
| 目标名称        | 必填   | 员工   | 非空，最长 200 字符                 |
| 关联指标        | 选填   | 员工   | 从指标库选择                       |
| 权重          | 必填   | 员工   | 整数，范围 1–100，所有目标权重合计须等于 100% |
| 目标描述        | 选填   | 员工   | 最长 2000 字符                   |
| 衡量标准        | 必填   | 员工   | 至少设置一个梯度值                    |
| 挑战值/目标值/保底值 | 必填   | 员工   | 仅量化指标，数值                     |
| 进度          | 选填   | 员工   | 数值，范围 0–100                  |

#### 字段校验与异常提醒

| 校验场景 | 提示内容 |
|---------|---------|
| 目标数量 < 2 | "目标数量不得少于 2 个" |
| 权重合计 ≠ 100% | "目标总权重必须等于 100%，当前为 XX%" |
| 必填字段为空 | 对应字段的必填提示 |
| 引用指标不存在 | "引用指标 [XX] 不存在或已失效" |
| 流程模板未配置 | "目标制定流程模板未配置，请联系管理员" |
| 并发冲突（他人已修改） | "数据已被他人修改，请刷新后重试" |
| 网络超时 | "提交超时，请检查网络后重试" |

#### 提交后的业务处理结果

1. 系统校验通过后，新增一条员工提交记录
2. 流程推进到双线上级审批节点
3. 审批人为空时自动跳过该节点
4. 直线上级审批通过后，目标状态变为"已审批"，抄送 HRBP
5. 目标生效数据 = 最近一次审批通过所引用的员工提交记录

#### 驳回/退回/重提规则

- 双线上级驳回：退回员工填写节点，员工修改后重新提交
- 直线上级驳回：退回节点由流程模板配置决定（可退双线上级或员工填写节点）
- 驳回后被驳回的员工提交记录标记为失效，审批动作记录保留

#### 状态变化

| 状态 | 含义 |
|------|------|

| 直线上级审批中 | 等待直线上级审批 |

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-1.1 | 员工提交时，目标数量 < 2 系统拒绝提交 |
| AC-1.2 | 员工提交时，权重合计 ≠ 100% 系统拒绝提交 |

| AC-1.7 | 批量导入时，部分失败行不进入流程，导入结果展示成功/失败条数 |

| AC-1.13 | 直线上级驳回后，流程退回 HRBP，HRBP 可重新导入并提交 |

### 4.2 绩效考核表单

#### 业务场景

绩效考核环节基于已审批通过的目标数据，由员工自评、双线上级评分、直线上级评分，最终计算指标得分、节点表单总分、最终得分和等级映射。计算结果单向流入结果审定环节（只读引用）。

| 角色 | 操作 |

| 双线上级 | 查看、评分、提交、驳回 |

#### 用户进入入口

> 原型参考：`perf-form-evaluation.html`。原型仅作交互参考。

#### 用户可见内容

**基本信息（只读）：** 被考核人、部门、职位、周期、模块名称、合计权重

**目标数据（只读）：** 指标名称、权重、描述、衡量标准、目标值、挑战值、保底值、进度

**评分数据：** 各节点的完成值、完成情况、评价结果、指标得分、评语、加减分

**计算结果（只读）：** 指标得分、节点表单总分、最终得分、最终等级

| 字段       | 是否必填     | 说明                   |

| 字段 | 是否必填 | 说明 |

| 校验场景 | 提示内容 |
|---------|---------|
| 量化指标完成值为空 | "量化指标完成值不能为空" |

| 总评语为空 | "总评语不能为空" |
| 目标生效记录不存在 | "目标数据不存在或已失效，请联系管理员" |

| 网络超时 | "提交超时，请检查网络后重试" |

#### 提交后的业务处理结果

**员工自评提交：**
1. 系统校验通过后，为每条指标计算得分（量化指标按计算规则，定性指标按评价规则等级分值）
2. 计算节点表单总分
3. 新增一条员工提交记录，记录计算过程和结果
4. 流程推进到双线上级评分节点

**双线上级/直线上级提交：**
1. 系统记录本节点的上级指标评价、总评语、初评等级
2. 不重新计算逐指标得分
3. 新增一条员工提交记录
4. 直线上级提交后，所有节点完成，系统计算最终得分和最终等级
5. 流程结束

#### 驳回/退回/重提规则

- 双线上级驳回：退到员工自评节点，员工修改完成值/完成情况/评价结果后重新提交
- 直线上级驳回：退到双线上级评分节点，双线上级重新处理
- 驳回后被驳回的员工提交记录标记为失效，审批动作记录保留

| 状态 | 含义 |

| 未开始 | 环节未开启 |
| 员工自评中 | 员工正在自评或已被驳回待重填 |
| 双线上级评分中 | 等待双线上级评分 |
| 直线上级评分中 | 等待直线上级评分 |
| 已完成 | 所有节点完成，最终得分和等级已生成 |
| 已驳回 | 流程中有节点驳回 |

#### 验收标准

| 编号 | 验收标准 |

| AC-2.1 | 量化指标填写完成值后，系统按计算规则自动计算指标得分 |

| AC-2.10 | 上级提交时不重新计算逐指标得分，仅记录本节点评价数据 |
| AC-2.11 | 批量导入时，部分失败行不进入流程，导入结果展示成功/失败条数 |
| AC-2.12 | HRBP 下载的考核结果模板包含当前考核组所有员工行和全部指标字段 |

| AC-2.14 | 导入预览中成功行和失败行明确区分，失败行展示失败原因 |
| AC-2.15 | 确认导入后，成功行进入直线上级审批节点，失败行不进入流程 |
| AC-2.16 | 直线上级驳回后，流程退回 HRBP，HRBP 可重新导入并提交 |

| 逐级上级 | 查看、评级、提交、驳回             |

**基本信息（只读）：** 被考核人、部门、职位、周期

**考核结果（只读）：** 审定前总分、审定前等级

**强制分布统计（只读）：** 各等级要求比例、当前比例、要求人数、当前人数

#### 用户可填写字段

| 字段 | 是否必填 | 填写角色 | 说明 |
|------|---------|---------|------|
| 调整等级 | 必填 | 直线上级/双线上级/逐级上级 | 下拉选择有效等级，可等于或不同于审定前等级 |
| 调整原因 | 条件必填 | 直线上级/双线上级/逐级上级 | 等级变更时必填 |

#### 字段校验与异常提醒

| 校验场景 | 提示内容 |
|---------|---------|
| 等级变更未填原因 | "调整等级时必须填写调整原因" |
| 强制分布强控超限 | "强制分布超限：[等级] 要求 [X]%，当前 [Y]%" |
| 强制分布弱控超限 | "强制分布超限：[等级] 要求 [X]%，当前 [Y]%，仍可提交" |
| 考核结果不存在 | "绩效考核结果不存在，请先完成考核环节" |
| 强制分布规则或流程模板未配置 | "强制分布规则或流程模板未配置" |
| 并发冲突 | "数据已被他人修改，请刷新后重试" |

#### 提交后的业务处理结果

1. 系统校验通过后，写入调整等级
2. 按方案强制分布规则重新计算各等级当前人数和占比
3. 新增一条员工提交记录
4. 等级为 A 时，流程自动生成 HRBP 审批待办
5. 所有节点完成后，审定结果写入活动结果表

#### 驳回/退回/重提规则

- 直线上级评级为首节点，不可驳回
- 双线上级驳回：退到直线上级评级 + 强制分布回滚
- 逐级上级驳回：按逐级驳回规则退到上一个可处理节点 + 强制分布回滚
- HRBP 驳回：按逐级驳回规则退到上一个可处理节点 + 强制分布回滚

#### 状态变化

| 状态       | 含义                      |
| -------- | ----------------------- |
| 未开始      | 环节未开启                   |
| 直线上级评级中  | 等待直线上级评级                |
| 双线上级评级中  | 等待双线上级评级                |
| 逐级上级评级中  | 等待逐级上级评级                |
| HRBP 审批中 | 等待 HRBP 审批（仅等级为 "远超预期"） |
| 已完成      | 所有节点完成                  |
| 已驳回      | 流程中有节点驳回                |

#### 验收标准

| 编号 | 验收标准 |
|------|---------|
| AC-3.1 | 等级变更时，调整原因必填，不填则提交失败 |
| AC-3.2 | 提交后强制分布计数更新，分母包含跳过审定和强制结束的员工 |
| AC-3.3 | 跳过审定/强制结束的员工使用方案配置默认等级，计入分母和对应等级分子 |
| AC-3.4 | 仍在审定流程中尚未产生有效等级的员工，只计入分母，不计入任何等级分子 |
| AC-3.5 | 强控模式下强制分布超限时，系统阻断提交 |
| AC-3.6 | 弱控模式下强制分布超限时，系统允许提交但返回警告 |
| AC-3.7 | 双线上级驳回后，退到直线上级评级，强制分布计数回滚 |
| AC-3.8 | HRBP 驳回 A 级审定后，按逐级驳回规则退到上一个可处理节点 |
| AC-3.9 | 等级为 A 时，流程自动生成 HRBP 审批待办 |
| AC-3.10 | 并发审定同一员工时，后提交者需等待 |

GHR → 绩效管理 → 考核活动 → 结果确认环节 → 员工确认表单

#### 用户可填写字段

| 字段 | 是否必填 | 填写角色 | 说明 |

| 编号 | 验收标准 |

| AC-5.5 | 面谈完成后，结果确认环节可实时读取面谈数据 |
| AC-5.6 | 面谈流程审批人为空时自动跳过 |

## 五、流程模型

### 5.1 目标制定流程

#### 通用 PBC 流程

| 顺序 | 节点 | 处理人 | 通过动作 | 驳回动作 | 跳过规则 |
|------|------|-------|---------|---------|---------|
| 1 | 目标填写 | 员工 | 进入双线上级审批 | — | — |
| 2 | 双线上级审批 | 双线上级 | 进入直线上级审批 | 退回员工填写 | 为空时跳过 |
| 3 | 直线上级审批 | 直线上级 | 目标生效，抄送 HRBP | 由流程模板配置决定退回节点 | 为空时跳过 |

#### 线上导入流程

| 顺序 | 节点 | 处理人 | 通过动作 | 驳回动作 | 跳过规则 |
|------|------|-------|---------|---------|---------|
| 1 | 目标导入 | HRBP | 进入直线上级审批 | — | — |
| 2 | 直线上级审批 | 直线上级 | 目标生效，抄送 HRBP | 退回目标导入（HRBP 重新导入） | 为空时跳过 |

### 5.2 绩效考核流程

| 顺序 | 节点 | 处理人 | 通过动作 | 驳回动作 | 跳过规则 |
|------|------|-------|---------|---------|---------|
| 1 | 员工自评 | 员工 | 进入双线上级评分 | — | — |
| 2 | 双线上级评分 | 双线上级 | 进入直线上级评分 | 退回员工自评（重新提交） | 为空时跳过 |
| 3 | 直线上级评分 | 直线上级 | 计算最终得分和等级，流程结束 | 退回双线上级评分 | 为空时跳过 |

**关键说明：**
- 节点 1 为首节点，不可驳回
- 员工自评提交时系统计算指标得分和节点表单总分
- 节点 2/3 提交时不重新计算逐指标得分，仅记录本节点评价数据
- 直线上级提交后，系统计算最终得分和最终等级

#### 考核结果线上导入流程

| 顺序 | 节点 | 处理人 | 通过动作 | 驳回动作 | 跳过规则 |
|------|------|-------|---------|---------|---------|
| 1 | 考核结果导入 | HRBP | 进入直线上级审批 | — | — |
| 2 | 直线上级审批 | 直线上级 | 考核结果生效，进入后续绩效流程 | 退回考核结果导入（HRBP 重新导入） | 为空时跳过 |

**关键说明：**
- 该流程为独立的线上导入考核流程，不走员工自评节点
- HRBP 导入并提交后，流程直接进入直线上级审批
- 直线上级审批通过后，考核结果生效
- 直线上级驳回时，退回 HRBP 节点，HRBP 可重新导入并提交
- 按行校验，部分行失败不阻断其他成功行的流程推进

### 5.3 结果审定流程

| 顺序 | 节点 | 处理人 | 通过动作 | 驳回动作 | 跳过规则 |
|------|------|-------|---------|---------|---------|
| 1 | 直线上级评级 | 直线上级 | 进入双线上级评级 | — | — |
| 2 | 双线上级评级 | 双线上级 | 进入逐级上级评级 | 退回直线上级评级 + 强制分布回滚 | 为空时跳过 |
| 3 | 逐级上级评级 | 逐级上级 | 等级为 A 进入 HRBP 审批；否则流程结束 | 按逐级驳回 + 强制分布回滚 | 为空时跳过 |
| 4 | HRBP 审批 | HRBP | 流程结束 | 按逐级驳回 + 强制分布回滚 | 等级 ≠ A 时跳过 |

**关键说明：**
- 节点 1 为首节点，不可驳回
- HRBP 审批仅在调整等级 = A（远超预期）时触发
- 逐级上级到配置的高管截止
- 驳回时强制分布计数回滚

### 5.4 结果确认流程

| 顺序 | 节点 | 处理人 | 通过动作 | 不同意动作 | 跳过规则 |
|------|------|-------|---------|-----------|---------|
| 1 | 员工确认 | 员工 | 流程结束，确认（同意） | 进入后续处理分支 | — |
| 2 | 后续处理 | 按流程配置（默认 HRBP/SSC/直线上级） | 流程结束，确认（不同意） | — | 员工同意时跳过 |

**关键说明：**
- 不支持驳回，员工操作只有同意或不同意
- 不同意后进入后续处理分支，后续处理节点由流程模板配置决定
- 超时自动确认：系统自动以"同意"身份完成
- 终态不可逆

### 5.5 绩效面谈流程

| 顺序 | 节点 | 处理人 | 通过动作 | 驳回动作 | 跳过规则 |
|------|------|-------|---------|---------|---------|
| 1 | 员工填写 | 员工 | 进入直线上级审批 | — | — |
| 2 | 直线上级审批 | 直线上级 | 面谈完成 | 退回员工填写 | 为空时跳过 |

> 本附录定义非功能性验收标准，用于产品验收和测试。

### B.1 响应体验

| 场景 | 验收标准 |
|------|---------|
| 表单加载 | 单人表单加载（100 条目标以内）≤ 2 秒 |
| 提交响应 | 单人提交操作 ≤ 3 秒 |
| 批量导入 | 1000 条数据导入 ≤ 30 秒 |
| 强制分布查询 | 实时计算，响应 ≤ 1 秒 |

### B.2 重复提交防护

| 场景 | 验收标准 |
|------|---------|
| 同一员工并发提交 | 后提交者收到"数据已被他人修改，请刷新后重试"提示 |
| 批量提交 | 按勾选项逐项校验，通过项允许提交，失败项不阻断其他项 |
| 强制分布并发更新 | 不出现计数不一致 |

### B.3 权限隔离

| 场景 | 验收标准 |
|------|---------|
| 员工操作范围 | 员工只能查看和操作自己的数据 |
| 上级操作范围 | 上级只能操作下级的数据 |
| 审批人数据范围 | 审批人只能查看被审批的员工提交内容 |

### B.4 操作留痕

| 场景 | 验收标准 |
|------|---------|
| 操作日志 | 所有提交、审批、确认操作记录操作人、操作时间、操作类型、操作结果 |
| 日志保留 | 至少保留 3 年 |
| 日志查询 | 支持按活动、员工、时间范围查询 |

### B.5 异常可追踪

| 场景 | 验收标准 |
|------|---------|
| 提交失败 | 返回明确的错误提示，用户可理解并修正 |
| 流程异常 | 标记当前节点为异常状态，不自动流转 |
| 配置缺失 | 阻止提交并提示"请联系管理员" |
| 引用配置失效 | 活动使用快照，不受影响 |
| 计算追溯 | 可查看每条指标的输入值、计算规则、得分 |
| 流程追溯 | 可查看完整流程节点历史（通过、驳回、跳过） |

### B.6 批量导入限制

| 指标 | 要求 |
|------|------|
| 文件大小 | 单文件 ≤ 10MB |
| 数据行数 | 单次 ≤ 5000 行 |
| 文件格式 | Excel (.xlsx) 或 CSV |
| 编码 | UTF-8 |

> 完整的批量导入规则（包括导入流程、校验原则、模板导出规则、操作留痕）见正文 3.10 节。各类导入的具体模板字段和校验规则见 4.1 节"目标批量导入"和 4.2 节"考核结果批量导入"。

## 十、本次产品化改写说明

### 改写目标

将原技术导向的 PRD 重构为产品导向的 PRD，使业务方、产品、测试都能直接按文档理解场景和验收。

### 主要变更

| 变更项 | 说明 |
|-------|------|
| 文档结构 | 按"业务场景 → 角色 → 用户任务 → 页面行为 → 字段规则 → 异常提醒 → 流程状态 → 验收标准"重构每个表单章节 |
| 第三章重写 | 从"技术职责边界"改为"通用业务规则"，包含正式提交、审批整表处理、驳回重提、评分计算、强制分布、跳过/强制结束、结果确认、权限可见性共 9 类规则 |

计算规则架构设计

衡量标准（参考值） × 计算规则（公式） × 指标库（绑定） — 从配置到执行

一、三者关系（核心设计）
核心原则：衡量标准只存参考值数据，不定义计算方式。计算规则只定义公式，不存参考值。指标库把两者绑定：这个指标用哪些参考值 + 用哪个公式算出得分。
📏 指标库（Indicator）— 绑定者

定义"考核什么"，同时绑定参考值和公式

  • 指标名称：季度营收达成率
  • 指标编码：KPI_001
  • 指标分类：业务目标/团队管理/能力/价值观
  • 计量单位：万元、次、%、分
  • FK → 衡量标准：参考值数据
  • FK → 计算规则：计算公式

定义"参考值"，纯数据，不含计算逻辑

  • 标准名称：营收类参考值
  • type: 百分比 / 分值 / 等级（仅分类标签）
  • grades 数组：
  • [{name, min, max, desc}]

💡 复用示例：同一参考值 + 不同公式 = 不同评分结果
场景A：营收达成率 — 梯度插值公式
// 衡量标准：挑战120, 目标100, 保底80 // 计算规则：梯度插值 // 员工完成值：115 得分 = gradient_interp( actual=115, baseline=80, target=100, challenge=120, target_score=100, max_score=120 ) // 115 在 [100,120] 区间 // = 100 + (115-100)/(120-100) × (120-100) = 115分
场景B：同一参考值 + 封顶公式
// 衡量标准：挑战120, 目标100, 保底80（同上） // 计算规则：封顶计算（超过挑战值不加分） // 员工完成值：130 得分 = capped_score( actual=130, baseline=80, target=100, challenge=120, cap=120 // 封顶值 = 挑战值 ) // 130 > 120（封顶） → 得分 = 120分（不超上限）
二、衡量标准 — 只存参考值，结构保持原型
衡量标准的结构与原型一致：{name, dept, type, grades:[{name, min, max, desc}]}。type 仅做分类标签（百分比/分值/等级），grades 存参考值区间。不包含任何计算逻辑。
字段类型说明示例
idbigint PK主键1
namevarchar(200)标准名称营收类参考值
deptvarchar(100)适用部门中国区
typeenum分类标签（非计算方式）百分比 / 分值 / 等级
gradesJSON参考值数组（核心数据）[{name:'卓越',min:100,max:100}, ...]
grades 数组在不同 type 下的含义
typegrades 含义示例
百分比 按完成百分比分段 [{name:'卓越',min:100,max:100}, {name:'优秀',min:90,max:99}, {name:'良好',min:75,max:89}, ...]
分值 按绝对分值分段 [{name:'非常满意',min:90,max:100}, {name:'满意',min:75,max:89}, ...]
等级 按等级名称分段 [{name:'A',min:110,max:120}, {name:'B',min:90,max:109}, ...]

💡 这些 grades 只是参考值。怎么用这些值算出得分，完全由计算规则的公式决定。

三、计算规则 — 只定义公式，可实现任意计算逻辑
计算规则的核心是公式表达式，由公式编辑器配置。公式可以引用衡量标准的参考值（grades）+ 表单字段（完成值/权重等），通过任意计算逻辑得出得分。
字段类型说明
idbigint PK主键
namevarchar(200)规则名称（如"梯度插值"、"封顶计算"）
formulatext公式表达式（公式编辑器生成）
formula_varsJSON公式引用的变量列表（字段白名单）
statusenumenabled / disabled
同一套参考值，套不同公式 = 不同计算结果
公式1：梯度插值
// 完成值在区间内线性插值 if actual >= challenge: score = max_score elif actual >= target: score = target_score + (actual-target)/(challenge-target) × (max_score-target_score) elif actual >= baseline: score = baseline_score + ... else: score = 0
公式2：封顶计算
// 超过挑战值不加分 if actual >= challenge: score = max_score // 封顶 elif actual >= target: score = target_score + (actual-target)/(challenge-target) × (max_score-target_score) else: score = actual/target × target_score
公式3：是否达成
// 只看是否达到目标值 if actual >= target: score = max_score // 满分 else: score = 0 // 零分

💡 以上三个公式都引用同一套参考值（grades），但计算结果完全不同。公式决定算法，参考值提供数据。

六、数据模型（配置层三张表）
📐 measure_standard
  • PK id bigint
  • name varchar(200)
  • dept varchar(100)
  • type enum(百分比/分值/等级)
  • grades JSON ← 只存参考值
grades=[{name,min,max,desc}]
📏 indicator
  • PK id bigint
  • code varchar(50) UNIQUE
  • name varchar(200)
  • category varchar(50)
  • unit varchar(20)
  • FK measure_std_id → 衡量标准
  • FK calc_rule_id → 计算规则 ← 绑定公式
🧮 calc_rule
  • PK id bigint
  • name varchar(200)
  • formula text ← 只存公式
  • formula_vars JSON
  • status enum(enabled/disabled)
SQL 关系
-- 指标库 → 衡量标准（多对一：多个指标可共用同一套参考值） indicator.measure_std_id → measure_standard.id -- 指标库 → 计算规则（多对一：多个指标可共用同一套公式） indicator.calc_rule_id → calc_rule.id -- 目标制定表单 → 指标库（员工选指标时带出参考值+公式） goal_detail.modules[*].goals[*].indicator_id → indicator.id -- 方案环节 → 计算规则（方案选择用哪套总分公式） perf_stage_config(stage='eval').calc_rule_id → calc_rule.id

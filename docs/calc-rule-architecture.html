计算规则架构设计

指标库 × 衡量标准 × 计算规则 × 公式编辑器 — 从配置到执行的完整链路

一、指标库 / 衡量标准 / 计算规则 的关系
📏 指标库（Indicator）

定义"考核什么" — 一个可度量的业务事项

  • 指标名称：如"季度营收达成率"
  • 指标编码：KPI_001（唯一标识）
  • 指标分类：业务目标/团队管理/能力发展/价值观
  • 计量单位：万元、次、%、分
  • 指标性质：定量 / 定性
  • 关联衡量标准：→ FK 衡量标准（决定怎么评分）
→
📐 衡量标准（MeasureStandard）

定义"怎么评分" — 指标得分的计算方式

  • 标准名称：如"营收类梯度评分"
  • 标准类型（4种）：
  • ① 梯度评分 目标值/挑战值/保底值 → 分数区间
  • ② 线性评分 线性插值计算得分
  • ③ 定性评分 评价结果(A/B/C/D/E) → 映射分数
  • ④ 是否达成 达成=满分/未达成=0分
→
🧮 计算规则（CalcRule）

定义"怎么汇总" — 多个指标得分→总分的聚合方式

  • 规则名称：如"加权求和"
  • 规则类型：
  • ① 加权求和 Σ(得分×权重/100)
  • ② 平均分 Σ得分/指标数
  • ③ 自定义公式 公式编辑器配置
  • 公式表达式：（自定义类型时由公式编辑器生成）

四、公式编辑器配置详解
📦 数据源定义（V1.1.1）
数据源 = 表单字段 + 系统字段

公式编辑器打开时，从以下数据源中选择可用字段：

数据源可用字段取数方式
目标制定表单 权重、目标值、挑战值、保底值 BOUND（绑定源字段）
考核表单 完成值、评价结果、加分、减分 BOUND（绑定源字段）
系统固定字段 指标得分、节点权重、目标数、模块权重 DYNAMIC（动态取数规则）
衡量标准参数 挑战值、目标值、保底值（来自指标库引用的标准） DYNAMIC（通过指标FK关联取数）
🔢 计算科目（V1.1.2）
科目 = 可参与公式计算的元素

两种科目类型：

科目类型说明示例
静态科目 BOUND 绑定到表单字段，值从表单数据中读取 完成值、权重、加分
动态科目 DYNAMIC 配置动态取数规则，运行时计算 指标得分、总分、等级映射分

可用函数：SUM、AVG、MAX、MIN、WEIGHTED_SUM、GRADE_MAP、ROUND

白名单控制（V1.1.3）
  • 应用白名单：哪些业务模块可以使用公式编辑器
  • 数据源级可见性：不同业务场景看到不同的字段列表
  • 取交集：最终可用字段 = 白名单 ∩ 数据源可见字段
💡 公式配置示例
示例1：标准加权求和（内置，无需公式编辑器）
// 规则类型：加权求和 // 系统自动执行，无需手动配置公式 总分 = WEIGHTED_SUM(指标得分, 权重) + 加分 - 减分 // 展开： 总分 = (指标1得分 × 指标1权重 / 100) + (指标2得分 × 指标2权重 / 100) + ... + 加分 - 减分
示例2：自定义公式（公式编辑器配置）
// 规则类型：自定义公式 // 在公式编辑器中拖拽字段+函数构建 综合得分 = WEIGHTED_SUM( GRADE_MAP(评价结果) × 权重 / 100, 节点权重 / 100 ) + 加分 - 减分 // GRADE_MAP 将定性评价映射为分数： // A→115, B→99.5, C→79.5, D→59.5, E→24.5 // WEIGHTED_SUM 按权重加权求和 // 最终结果 ROUND 到1位小数
示例3：梯度指标得分计算（衡量标准驱动）
// 衡量标准类型：梯度评分 // 配置：挑战值=120万, 目标值=100万, 保底值=80万 if (完成值 >= 挑战值) then 指标得分 = 指标满分 // 120分 else if (完成值 >= 目标值) then 指标得分 = 目标分数 + (完成值 - 目标值) / (挑战值 - 目标值) × (满分 - 目标分数) else if (完成值 >= 保底值) then 指标得分 = 保底分数 + (完成值 - 保底值) / (目标值 - 保底值) × (目标分数 - 保底分数) else 指标得分 = 0 // 低于保底值
示例4：定性指标得分（评价结果映射）
// 衡量标准类型：定性评分 // 上级选择评价结果 → 系统映射为分数 指标得分 = GRADE_MAP(评价结果) // 映射表： // "远超预期" → 115分 (110-120中值) // "超出预期" → 99.5分 (90-109中值) // "符合预期" → 79.5分 (70-89中值) // "需改进" → 59.5分 (50-69中值) // "远低于预期" → 24.5分 (0-49中值) // 总评价结果 = 节点权重加权 总评价结果 = WEIGHTED_SUM( 自评评价结果 × 自评节点权重, 间接上级评价结果 × 间接上级节点权重, 直接上级评价结果 × 直接上级节点权重 )

六、数据模型补充（配置层）
📐 measure_standard（衡量标准）
  • PK id bigint
  • name varchar(200) 标准名称
  • type enum(gradient/linear/qualitative/passfail)
  • config JSON（类型相关配置）
  • 梯度：{target,challenge,baseline,scores}
    定性：{A:115,B:99.5,...}
📏 indicator（指标库）
  • PK id bigint
  • code varchar(50) UNIQUE
  • name varchar(200)
  • category varchar(50) 分类
  • unit varchar(20) 计量单位
  • nature enum(quantitative/qualitative)
  • FK measure_std_id → measure_standard
🧮 calc_rule（计算规则）
  • PK id bigint
  • name varchar(200)
  • type enum(weighted_sum/avg/custom)
  • formula text（自定义公式表达式）
  • formula_vars JSON（公式引用的变量列表）
  • status enum(enabled/disabled)
🔗 关系总结
-- 指标库 → 衡量标准（多对一） indicator.measure_std_id → measure_standard.id -- 一个衡量标准可以被多个指标引用 -- 目标制定表单 → 指标库（多对一） goal_detail.modules[*].goals[*].indicator_id → indicator.id -- 员工选择指标时，自动带出衡量标准和计算规则 -- 绩效方案环节 → 计算规则（多对一） perf_stage_config(stage_type='eval').calc_rule_id → calc_rule.id -- 方案绩效考核环节选择用哪套计算规则 -- 绩效方案环节 → 评价规则（多对一） perf_stage_config(stage_type='eval').eval_rule_id → eval_rule.id -- 评价规则定义评价模式（上级评价/360等） -- 公式编辑器 → 计算规则（一对一，自定义类型时） calc_rule(type='custom').formula -- 由公式编辑器生成的表达式 calc_rule.formula_vars -- 公式引用的变量（字段白名单）

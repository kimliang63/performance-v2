## 1. 系统异常

### EX-001 网络故障 — 保存

| 项目 | 内容 |
|------|------|
| 场景 | 员工填写表单点击"保存草稿"时网络中断 |
| 触发条件 | HTTP 请求超时或连接失败 |
| 处理方式 | 1. 弹窗提示"网络异常，保存失败，请稍后重试" 2. 本地数据保留在浏览器（不丢失） 3. 恢复网络后可重新保存 |
| 数据影响 | 无数据丢失，草稿状态不变 |
| 用户感知 | 弹窗警告 |

### EX-002 网络故障 — 提交

| 项目 | 内容 |
|------|------|
| 场景 | 员工点击"提交"时网络中断 |
| 触发条件 | HTTP 请求超时或连接失败 |
| 处理方式 | 1. 弹窗提示"提交失败，请检查网络后重试" 2. 数据保留在当前编辑状态 3. 不触发流程流转 |
| 数据影响 | 无数据丢失，表单状态不变 |
| 用户感知 | 弹窗警告 |

### EX-003 会话超时

| 项目 | 内容 |
|------|------|
| 场景 | 用户长时间未操作，会话过期 |
| 触发条件 | 会话 TTL 到期 |
| 处理方式 | 1. 自动保存当前草稿（如有变更） 2. 弹窗提示"会话已过期，请重新登录" 3. 跳转到登录页 |
| 数据影响 | 已自动保存的草稿不丢失 |
| 用户感知 | 跳转登录页 |

### EX-004 服务器 5xx 错误

| 项目 | 内容 |
|------|------|
| 场景 | 任何写操作（保存/提交/审批）时服务器返回 5xx |
| 触发条件 | 服务端异常 |
| 处理方式 | 1. 弹窗提示"系统繁忙，请稍后重试" 2. 记录错误日志（含请求参数） 3. 数据保留当前状态 |
| 数据影响 | 无数据丢失 |
| 用户感知 | 弹窗警告 |

## 2. 并发异常

### EX-010 审定表单同时编辑

| 项目 | 内容 |
|------|------|
| 场景 | 直接上级和间接上级同时打开同一员工的审定表单编辑 |
| 触发条件 | 两个审批人同时操作 |
| 处理方式 | 1. 后提交者检测到数据版本冲突 2. 弹窗提示"该表单已被其他审批人处理，请刷新后查看最新状态" 3. 后提交者不可覆盖前序操作 |
| 数据影响 | 后提交者的修改被丢弃 |
| 用户感知 | 弹窗警告 + 强制刷新 |

### EX-011 PBC 编辑与审批并发

| 项目 | 内容 |
|------|------|
| 场景 | 员工正在编辑 PBC 目标，上级同时驳回了上一轮审批 |
| 触发条件 | 编辑操作和审批操作并发 |
| 处理方式 | 1. 驳回操作优先：员工编辑界面自动刷新为驳回后的最新状态 2. 弹窗提示"您的目标已被驳回，请查看最新状态" 3. 员工已编辑的未保存内容需要重新填写 |
| 数据影响 | 员工未保存的编辑内容可能丢失 |
| 用户感知 | 弹窗通知 + 界面刷新 |

### EX-012 批量导入与单条编辑冲突

| 项目 | 内容 |
|------|------|
| 场景 | HRBP 正在批量导入目标，员工同时在编辑同一批目标的单条记录 |
| 触发条件 | 批量操作和单条操作操作同一批数据 |
| 处理方式 | 1. 批量导入操作锁定目标数据 2. 员工单条编辑时检测到锁定，提示"当前有批量操作进行中，请稍后重试" 3. 导入完成后员工可继续编辑 |
| 数据影响 | 单条编辑被阻断，不丢失 |
| 用户感知 | 操作被阻止 + 提示信息 |

## 3. 权限异常

### EX-020 角色变更 — 审批中

| 项目 | 内容 |
|------|------|
| 场景 | 审批流程进行中，审批人的角色/职位发生变更（如调岗、离职） |
| 触发条件 | 汇报关系或组织角色变更 |
| 处理方式 | 1. 当前节点：如审批人仍有效，继续；如无效，自动跳过 2. 未到达节点：按最新汇报关系重新查询审批人 3. 已完成节点：不回退 |
| 数据影响 | 已完成审批不回退 |
| 用户感知 | 可能自动跳过某节点 |

### EX-021 组织架构调整 — 审定中

| 项目 | 内容 |
|------|------|
| 场景 | 组织架构逐级审定进行中，组织树结构发生调整 |
| 触发条件 | 组织合并/拆分/层级调整 |
| 处理方式 | 1. 逐级上级查询按调整后的组织架构执行 2. 如原审批人不在新组织中，自动跳过 3. 如新组织导致审定路径变化，按新路径继续 |
| 数据影响 | 审定路径可能变化 |
| 用户感知 | 审定流向可能改变 |

### EX-022 错误周期访问

| 项目 | 内容 |
|------|------|
| 场景 | 用户尝试访问已关闭的绩效周期的表单 |
| 触发条件 | 周期已过截止时间 |
| 处理方式 | 1. 只读模式展示历史数据 2. 编辑/提交操作全部禁用 3. 提示"该绩效周期已结束" |
| 数据影响 | 无 |
| 用户感知 | 只读展示 |

## 4. 业务异常

### EX-030 确认驳回 — 无闭环

| 项目 | 内容 |
|------|------|
| 场景 | 员工确认"不同意"后，HRBP 知悉，但线下处理无进展 |
| 触发条件 | 员工点击"不同意" → HRBP 知悉 → 无后续动作 |
| 处理方式 | 1. HRBP 知悉后流程状态为 `completed`（终态） 2. 系统不发起后续流程 3. 线下处理由 HRBP 手动跟进 |
| 数据影响 | 流程已结束，无系统内闭环 |
| 用户感知 | 员工看到"已提交异议"状态 |

### EX-031 HRBP 驳回审定

| 项目 | 内容 |
|------|------|
| 场景 | HRBP 在审定环节驳回（调整后等级=A 时触发的审批） |
| 触发条件 | HRBP 点击"不同意" |
| 处理方式 | 1. 审定流程回到 `pending_direct_rate` 状态 2. 直接上级重新评级 3. 重新走完整审定流程 |
| 数据影响 | 审定数据重置到初始审定状态 |
| 用户感知 | 审定流程重新开始 |

### EX-032 全空审批链

| 项目 | 内容 |
|------|------|
| 场景 | 员工的所有上级（间接+直接）均为空 |
| 触发条件 | 汇报关系中无任何上级配置 |
| 处理方式 | 1. 间接上级节点自动跳过 2. 直接上级节点自动跳过 3. 流程直接到达终态 4. 系统记录异常日志 |
| 数据影响 | 表单数据直接生效，无审批 |
| 用户感知 | 流程快速结束 |

### EX-033 导入目标异议

| 项目 | 内容 |
|------|------|
| 场景 | HRBP 批量导入目标后，员工点击"不同意" |
| 触发条件 | 员工在 `pending_employee_confirm` 状态选择不同意 |
| 处理方式 | 1. 流程回到 `goal_import` 状态 2. HRBP 重新导入 3. 员工再次确认 |
| 数据影响 | 原导入数据被覆盖 |
| 用户感知 | HRBP 收到退回通知 |

### EX-034 批量导入部分失败

| 项目 | 内容 |
|------|------|
| 场景 | HRBP 批量导入 100 条目标，其中 5 条校验失败 |
| 触发条件 | 导入数据中存在校验不通过的记录 |
| 处理方式 | 1. 成功的 95 条正常导入 2. 失败的 5 条返回错误明细（行号 + 原因） 3. 弹窗展示失败列表 4. 用户可修正后重新导入失败的记录 |
| 数据影响 | 成功记录不受影响 |
| 用户感知 | 导入结果弹窗（成功/失败明细） |

### EX-035 强控死锁

| 项目 | 内容 |
|------|------|
| 场景 | 强控模式下，所有等级的实际人数均已达到上限，无法再调整 |
| 触发条件 | 强控超限 + 无可用等级槽位 |
| 处理方式 | 1. 审定表单提交按钮禁用 2. 弹窗提示"强制分布已满，请调整其他员工的等级" 3. 管理员需先调整其他员工等级释放槽位 |
| 数据影响 | 审定流程阻断 |
| 用户感知 | 提交被阻止 + 弹窗提示 |

## 5. 数据完整性异常

### EX-040 等级规则修改 — 审定中

| 项目 | 内容 |
|------|------|
| 场景 | 管理员在审定流程进行中修改了等级规则（如调整分数区间） |
| 触发条件 | 方案配置变更 |
| 处理方式 | 1. 已进行的审定不受影响（使用修改前的规则） 2. 未进行的审定使用新规则 3. 方案修改需在活动启动前完成 |
| 数据影响 | 新旧规则并存 |
| 用户感知 | 无感知 |

### EX-041 指标删除 — 目标引用中

| 项目 | 内容 |
|------|------|
| 场景 | 管理员从指标库删除某指标，但该指标已被员工目标引用 |
| 触发条件 | 指标库变更 |
| 处理方式 | 1. 已引用的目标不受影响（保留引用快照） 2. 删除的指标从可选列表中移除 3. 新目标不可引用已删除指标 |
| 数据影响 | 历史引用保留 |
| 用户感知 | 新目标选择时看不到已删除指标 |

### EX-042 员工调岗 — 考核中

| 项目 | 内容 |
|------|------|
| 场景 | 员工在考核流程进行中发生调岗（部门/汇报关系变更） |
| 触发条件 | 人事系统同步调岗信息 |
| 处理方式 | 1. 当前流程按原部门/汇报关系继续 2. 调岗后的新流程在新部门发起 3. 不影响当前流程的审批链 |
| 数据影响 | 原流程不受影响 |
| 用户感知 | 可能在新旧部门各有一个流程 |

### EX-043 汇报关系变更 — 审批链断裂

| 项目 | 内容 |
|------|------|
| 场景 | 审批链进行中，某审批人的汇报关系发生变更 |
| 触发条件 | 汇报关系调整 |
| 处理方式 | 1. 已完成节点：不回退 2. 当前节点：如审批人仍有效，继续；如无效，自动跳过 3. 未到达节点：按新汇报关系查询 |
| 数据影响 | 已完成审批不回退 |
| 用户感知 | 审批链可能跳过某节点 |

## 6. 超时与自动保存

### EX-050 自动保存

| 项目 | 内容 |
|------|------|
| 场景 | 用户填写表单过程中 |
| 触发条件 | 每 30 秒自动触发，或字段变更后 5 秒防抖 |
| 处理方式 | 1. 自动保存草稿到本地 + 服务端 2. 不触发流程流转 3. 保存失败静默重试（最多 3 次） |
| 数据影响 | 草稿数据持续更新 |
| 用户感知 | 右下角 toast 提示"已自动保存" |

### EX-051 活动级超时 — 待 V4 定义

| 项目 | 内容 |
|------|------|
| 场景 | 活动配置了截止时间，某环节超时未完成 |
| 触发条件 | 当前时间 > 活动截止时间 |
| 处理方式 | **待 V4 定义**。默认假设：1. 发送催办通知 2. 超过截止时间 N 天后自动结束该环节 3. 标记为异常 |
| 数据影响 | 环节可能被强制结束 |
| 用户感知 | 催办通知 + 状态变更 |

### EX-052 确认超时

| 项目 | 内容 |
|------|------|
| 场景 | 员工确认环节超时未操作 |
| 触发条件 | 确认环节持续超过配置天数 |
| 处理方式 | 1. 超时后系统自动确认为"同意" 2. 发送通知告知员工已自动确认 3. 流程正常结束 |
| 数据影响 | 确认状态变为 completed |
| 用户感知 | 收到自动确认通知 |

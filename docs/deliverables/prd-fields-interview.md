# 绩效面谈表单 — 字段清单与交互规则

> 基于 `perf-form-interview.html` 原型提取，版本 v3 快照

---

## 1. 页面布局

```
┌─────────────────────────────────────────────────────────┐
│ Header: Logo + 返回导航 + 标题"结果面谈｜员工本人填写"   │
├─────────────────────────────────────────────────────────┤
│ Page Header: 标题 + 考核周期 + 状态 badge               │
├─────────────────────────────────────────────────────────┤
│ Employee Selector: 切换不同员工的面谈表单               │
├─────────────────────────────────────────────────────────┤
│ 基本信息: 考核周期 / 发起日期 / 被考核人 / 部门 /       │
│          职位 / 直接上级（全部只读）                     │
├─────────────────────────────────────────────────────────┤
│ 面谈内容:                                               │
│   ├ 是否进行过绩效面谈（select）                       │
│   ├ 点三个赞（textarea）                               │
│   ├ 拍三个转（textarea）                               │
│   ├ 员工下一阶段规划（textarea）                       │
│   ├ 提一个需求（textarea）                             │
│   └ 附件（文件上传）                                   │
├─────────────────────────────────────────────────────────┤
│ 上级评语:                                               │
│   └ 上级评语（textarea）                               │
├─────────────────────────────────────────────────────────┤
│ Actions: 保存草稿/提交 或 驳回/同意 或 已完成badge     │
└─────────────────────────────────────────────────────────┘
```

## 2. 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|

## 4. 面谈内容区字段

| 字段名 | 控件 | 类型 | 必填 | 校验 | 说明 |
|--------|------|------|------|------|------|
| 您的直线上级是否有跟您进行过年度绩效面谈？ | select | enum | 是 | 必选 | 选项：是，一对一面谈 / 是，电话面谈 / 是，其他形式面谈 / 否 |

## 7. 页面交互逻辑汇总

### 7.1 基础操作

| 场景 | 行为 |
|------|------|
| 切换员工 | 点击顶部员工按钮 → 加载对应面谈表单 |
| 保存草稿 | 保存当前数据，不校验，toast"草稿已保存" |
| 员工提交 | 校验必填项 → 通过后状态流转为 pending_leader_review |
| 上级同意 | 校验评语 → 通过后状态流转为 completed |
| 上级驳回 | 必须填写评语 → 状态流转回 pending_employee_fill |
| 拖拽上传 | 支持多文件，可删除已上传文件 |

### 7.2 状态驱动 UI

| 状态 | UI 表现 |
|------|---------|
| `not_started` | 空状态提示"面谈尚未开始" |
| `pending_employee_fill` | 面谈内容可编辑，上级评语只读，底部显示"保存草稿"+"提交" |
| `pending_leader_review` | 面谈内容只读，上级评语可编辑，底部显示"驳回"+"同意" |
| `completed` | 所有字段只读，显示绿色"面谈已完成"badge |

### 7.3 操作反馈

| 操作 | 反馈方式 | 内容 |
|------|----------|------|
| 保存草稿 | toast（底部，2.5秒消失） | "草稿已保存" |
| 提交成功 | toast + 状态变更 | "提交成功" |
| 提交失败 | toast（红色） | 具体校验错误信息 |
| 审批通过 | toast + 状态变更 | "审批通过" |
| 驳回 | toast + 状态变更 | "已驳回，员工需重新填写" |

### 7.4 跨页面联动

| 触发事件 | 联动行为 |
|----------|----------|
| 员工提交 | 直接上级收到待办通知 |
| 上级同意 | 考核流程结束，触发确认环节 |
| 上级驳回 | 员工收到待办通知，需重新填写 |

### 7.5 异常态交互

| 异常场景 | UI 表现 | 用户操作 |
|----------|---------|----------|
| 网络断开保存 | toast 红色"保存失败" | 数据保留在本地 |
| 网络断开提交 | toast 红色"提交失败" | 数据保留，不触发流转 |
| 会话超时 | 弹窗"会话已过期" → 跳转登录 | 重新登录后回到表单 |

### 7.6 Loading 态

| 场景 | 表现 |
|------|------|
| 页面初始化 | 表单区域显示骨架屏 |
| 切换员工 | 右侧表单区域显示 loading spinner |
| 保存/提交中 | 按钮显示 spinner + 禁用 |

## 8. 说明

- 面谈内容字段名称与 Axure 参考原型保持一致：点三个赞、拍三个转、员工下一阶段规划、提一个需求
- "您的直线上级是否有跟您进行过年度绩效面谈？"为面谈前提确认，选择"否"时仍需填写面谈内容
- 上级评语仅在上级审批阶段可编辑，员工填写阶段和完成后均为只读
- 面谈表单与确认表单存在字段重叠（FIXME #5），待业务确认后明确数据归属

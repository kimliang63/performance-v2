# 绩效考核表单 — 字段清单与交互规则

> 基于 `perf-form-evaluation.html` 原型提取，版本 v3 快照

---

## 1. 页面布局

### 1.1 两种模式

| 模式 | 布局 | 说明 |
|------|------|------|
| 单人评分 | 平铺卡片：基本信息 + 指标表 + 底部总评 | 逐个评分 |
| 批量对比 | 左右结构：左侧员工列表 + 右侧表单 | 对比评分，支持批量导入 |

### 1.2 整体结构

```
┌─────────────────────────────────────────────────────────┐
│ Header: 面包屑 + 流程步骤条 + 模式切换（单人/批量）      │
├─────────────────────────────────────────────────────────┤
│ Page Header: 标题 + 操作按钮（暂存/提交/模板导出/批量导入）│
├──────────────────────┬──────────────────────────────────┤
│ 单人模式:            │ 批量模式:                         │
│ 基本信息网格          │ 左侧: 搜索+筛选+员工列表         │
│ 指标评分表            │ 右侧: 员工考核表单               │
│ 底部: 总分+总评语     │ 底部: 总分+总评语+提交            │
└──────────────────────┴──────────────────────────────────┘
```

## 2. 数据结构

### 2.1 Indicator 对象（考核指标）

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 指标名称 |
| `type` | `'quant' \| 'qual' \| 'bonus'` | 定量/定性/加减分 |

| `bonusType` | `'bonus' \| 'penalty'` | 加分/减分 |

### 2.2 Employee 对象

| 字段 | 类型 | 说明 |

| `id` | string | 工号 |

## 5. 底部区域字段

| 字段名 | 控件 | 类型 | 必填 | 校验 | 说明 |
|--------|------|------|------|------|------|
| 加权总分 | 文本 | number | — | — | 自动计算，只读 |
| 总评语 | textarea | string | 否 | 0-1000字符 | |

### 加权总分计算逻辑

```
加权总分 = Σ(评价分值 × 指标权重/100) + 加减分合计
```

- 定量指标：需输入完成值后由系统自动计算得分（原型中未实现自动计算，需人工填等级）
- 定性指标：选择等级后直接映射分值
- 加减分：直接累加

## 9. 提交校验规则

### 单人提交

| 校验项 | 规则 | 错误提示 | 阻断 |

| 总分 | 必须 > 0 | "请先完成评分再提交" | 是（toast） |

## 10. 页面交互逻辑汇总

### 10.1 基础操作

| 场景 | 行为 |
|------|------|
| 切换单人/批量模式 | 顶部模式切换按钮，切换显示对应区域 |
| 单人模式选择员工 | 左侧列表点击 → 右侧加载该员工表单 |
| 评分输入（定量） | 输入完成值 → 实时更新总分 |
| 评分输入（定性） | 选择等级 → 实时更新总分 |
| 加减分类型切换 | 切换加分/减分 → 自动调整分值正负号 |
| 暂存 | 保存当前数据，不校验，toast 提示"表单已保存为草稿" |
| 提交 | 校验总分 > 0 → 通过则提交，不通过 toast 提示 |
| 模板导出 | 生成 CSV 文件下载 |
| 打开导入弹窗 | 隐藏上传区，显示预览表（模拟） |
| 拖拽上传 | 拖拽文件到上传区 → 触发导入流程 |
| 确认导入 | 更新员工评分数据，刷新列表 |
| 批量提交 | 提交所有已评分员工，toast 提示 |
| 搜索员工 | 按姓名/部门模糊搜索 |
| 筛选员工 | "全部"/"待评"切换 |

### 10.2 状态驱动 UI

| 状态 | UI 表现 |
|------|---------|
| `self_scoring`（员工自评） | 员工可编辑：完成值、各指标评价、评语；总评价结果/总评语只读 |
| `pending_indirect_scoring` | 间接上级可编辑：各指标评价、评语；完成值只读；总评价结果/总评语只读 |
| `pending_direct_scoring` | 直接上级可编辑：各指标评价、评语、**总评价结果、总评语**；完成值只读 |
| `completed` | 所有字段只读；显示绿色"已完成"状态标签 |

### 10.3 字段权限矩阵

| 字段 | 自评(节点1) | 间接上级(节点2) | 直接上级(节点3) |
|------|-----------|----------------|----------------|
| 完成值 | 可编辑/必填 | 只读 | 只读 |
| 评价结果(各指标) | 可编辑/必填 | 可编辑/必填 | 可编辑/必填 |
| 评语 | 可编辑 | 可编辑 | 可编辑 |
| 总评价结果 | 只读 | 只读 | **可编辑/必填** |
| 总评语 | 只读 | 只读 | **可编辑** |

### 10.4 操作反馈

| 操作 | 反馈方式 | 内容 |
|------|----------|------|
| 保存草稿 | toast（底部，3秒消失） | "草稿已保存" |
| 提交成功 | toast + 页面状态变更 | "评分已提交" |
| 提交失败（校验不通过） | toast（红色） | 具体校验错误信息 |
| 批量提交成功 | toast | "已提交 N 人评分，M 人失败" |
| 批量提交部分失败 | toast + 失败明细弹窗 | 弹窗列出失败员工及原因 |
| 评分保存中 | 按钮 loading 态 | 按钮显示 spinner，禁用重复点击 |

### 10.5 跨页面联动

| 触发事件 | 联动行为 |
|----------|----------|
| 考核完成（`completed`） | 审定表单自动创建，考核数据写入审定表单只读区 |
| 员工自评提交 | 间接上级收到待办通知 |
| 间接上级评分提交 | 直接上级收到待办通知 |
| 直接上级评分提交 | 考核流程结束，触发审定环节 |

### 10.6 异常态交互

| 异常场景 | UI 表现 | 用户操作 |
|----------|---------|----------|
| 网络断开保存 | toast 红色"保存失败" | 数据保留在本地 |
| 网络断开提交 | toast 红色"提交失败" | 数据保留，不触发流转 |
| 会话超时 | 弹窗"会话已过期" → 跳转登录 | 重新登录后回到表单 |
| 并发编辑（上级同时评分） | 弹窗"该表单已被其他审批人处理，请刷新" → 强制刷新 | 查看最新状态 |
| 总分 ≤ 0 | toast 红色"请先完成评分再提交" | 继续编辑 |
| 批量导入文件格式错误 | 弹窗"文件格式不支持，请上传 Excel 文件" | 重新选择文件 |
| 批量导入部分失败 | 弹窗列出失败行号+原因 | 修正后重新导入 |

### 10.7 Loading 态

| 场景 | 表现 |
|------|------|
| 页面初始化 | 表单区域显示骨架屏 |
| 切换员工 | 右侧表单区域显示 loading spinner |
| 保存/提交中 | 按钮显示 spinner + 禁用 |
| 导入处理中 | 上传区显示进度条 |
| 批量提交中 | 提交按钮显示 spinner + 禁用 |

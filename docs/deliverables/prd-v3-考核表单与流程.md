## 一、需求概述

### 1.1 需求背景

1. V2 绩效方案定义了考核框架（考核组+环节+流程+表单关联），V3 需要将方案中关联的表单和流程进行完整定义；
2. 5类考核表单（PBC目标制定、绩效考核、结果审定、员工确认、绩效面谈）的字段、处理方式、权限需逐一定稿；
3. 各环节对应的流程模型（节点、分支、审批人设置、抄送）需完成配置清单。

### 1.2 目标用户

| 角色名称 | 用户场景 | 用户故事 |
|----------|----------|----------|
| 被考核人 | 填写个人目标、确认绩效结果、填写面谈记录 | 我需要在系统中完成目标制定、查看考核结果、提交面谈记录 |
| 考核人 | 对被考核人进行绩效评分 | 我需要查看下属目标、逐个或批量评分、填写评语 |
| 审定人 | 审定绩效结果，调整等级 | 我需要查看部门绩效分布、调整等级、提交审定结果 |
| HRBP | 监控考核进度，处理员工申诉 | 我需要查看考核进展、处理员工确认驳回 |

### 1.3 产品架构图

{V3 在 V2 方案层之下增加表单执行层和流程引擎层}

### 1.4 产品结构图

```
GHR / 绩效管理
├── 绩效方案（V2）→ 关联 ↓
├── 考核表单（V3）
│   ├── PBC目标制定表单
│   ├── 绩效考核表单
│   ├── 结果审定表单
│   ├── 员工确认表单
│   └── 绩效面谈表单
├── 绩效流程（V3）
│   ├── 目标制定流程
│   ├── 绩效考核流程
│   ├── 结果审定流程
│   ├── 结果确认+申诉流程
│   └── 面谈流程
└── 流程引擎平台（独立模块）
    ├── 流程设计器
    ├── 流程管理
    └── 表单管理
```

## 二、需求目标

1. 完成5类考核表单的字段定义、处理方式、权限矩阵
2. 完成5个绩效流程的节点定义、分支规则、审批人设置
3. 表单与流程的关联关系明确
4. 表单支持单人处理、批量对比、批量导入等多种操作模式

#### 4.14 绩效流程操作

| 操作 | 说明 |
|------|------|
| 提交（通过） | 正向操作，流转到下一节点 |
| 转交 | 将当前任务转交给其他人处理 |
| 驳回 | 退回上一节点或发起人 |
| 撤回 | 发起人撤回已提交的任务 |
| 催办 | 对超时未处理的节点发送催办通知 |

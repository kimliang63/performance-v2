## 已确认信息

| 序号 | 问题 | 结论 |
|------|------|------|
| 1 | 间接上级为空时的处理方式 | 为空时自动跳过 |
| 2 | 双线上级为空时的处理方式 | 为空时自动跳过 |
| 3 | 直接上级为空时的处理方式 | 为空时自动跳过 |
| 4 | 结果审定停止人员 | 流程配置配特定人结束，用户自己输入 |
| 5 | 结果确认处理人 | 被考核人所在组织架构的HRBP |
| 6 | 审定模式切换条件 | 用户在绩效方案配置时自己勾选 |
| 7 | 导入流程数据来源 | Excel导入 |
| 8 | 间接上级 vs 双线上级 | 间接上级=直接上级的直接上级（汇报线）；双线上级=花名册独立维护 |
| 9 | HRBP在结果确认环节的处理 | 知悉→结束，不驳回员工，不确认结果线下处理 |
| 10 | 跳过审定员工的绩效结果 | 绩效方案中定义固定绩效结果等级 |
| 11 | 间接/直接上级多人审批 | 系统设计只有一个人或为空，不存在多人审批场景 |
| 12 | HRBP为空时的处理方式 | 跳过 |
| 13 | 面谈与确认的先后顺序 | 确认永远在最后 |
| 14 | 等级映射规则 | 基础设置版本已定义计算规则 |

## 关键术语澄清

| 术语 | 定义 | 来源 |
|------|------|------|
| 间接上级 | 直接上级的直接上级 | 汇报关系 |
| 双线上级 | 员工的另一个上级 | 花名册独立维护 |
| 直接上级 | 员工的直接汇报上级 | 汇报关系 |
| 逐级上级 | 汇报线/组织架构逐级向上 | 汇报关系或组织架构 |

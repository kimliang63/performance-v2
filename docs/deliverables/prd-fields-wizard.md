## 1. 页面布局

### 1.1 整体结构

```
┌─────────────────────────────────────────────────────────┐
│ Header: Logo + 标题 + [取消] [保存] 按钮                │
├──────────┬──────────────────────────────────────────────┤
│ 侧边栏   │ 内容区                                       │
│ (220px)  │                                              │
│          │ 按 activeSection 渲染不同内容：               │

│ ├ 基本信息│  ├─ groups → 考核组配置                     │

│ └ 考核环节│  └─ goal/eval/ratify/... → 环节详情         │
│   ├ 目标  │                                             │
```

## 2. 数据结构

### 2.1 form 对象（方案配置）

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 方案名称 |
| `code` | string | 方案编号，自动生成 |
| `cycle` | string | 周期类型：月度/半年度/年度/自定义 |
| `enabled` | boolean | 是否启用 |
| `downward` | boolean | 是否向下公开 |
| `desc` | string | 规则描述 |
| `dept` | string | 适用部门 |
| `gradeRule` | string | 引用的等级规则名称 |
| `grades` | array | 等级列表（从规则带入） |
| `stages` | Stage[] | 考核阶段配置 |
| `groups` | Group[] | 考核组配置 |

### 2.2 Stage 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 阶段ID |
| `name` | string | 阶段名称 |
| `desc` | string | 阶段描述 |
| `enabled` | boolean | 是否启用 |
| `status` | string | 状态：done/warn/active/pending |
| `flow` | string | 关联的流程模型名称 |

### 2.3 Group 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 考核组ID |
| `name` | string | 考核组名称 |
| `rules` | Rule[] | 适用规则数组 |
| `excluded` | string[] | 排除人员名单 |
| `evalRule` | string | 评价规则名称 |

### 2.4 Rule 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `field` | string | 筛选字段：部门/职位/职级/人员/入职时间/用工类型 |
| `op` | string | 操作符：等于/不等于/包含/属于 |
| `val` | string | 筛选值 |

### 2.5 GRADE_RULES（等级规则库）

| 规则名称 | 等级 |
|----------|------|
| 标准五级 | 远超预期(A) / 符合预期+(B+) / 符合预期(B) / 符合预期-(B-) / 低于预期(C) |
| 简化三级 | 优秀(A) / 合格(B) / 待改进(C) |
| 四级制 | 卓越(S) / 优秀(A) / 合格(B) / 不合格(C) |

### 2.6 TEMPLATES（模板预设）

| 周期 | 启用阶段 | 需要等级 |
|------|----------|----------|
| 月度 | goal, eval, ratify, confirm | 是 |
| 半年度 | interview | 否 |
| 年度 | interview, ratify | 是 |

### 2.7 FLOW_MODELS（流程模型）

| 阶段 | 可选流程模型 |
|------|-------------|
| goal | 通用目标制定流程 / 简化目标制定流程 |
| eval | 通用绩效考核流程 / 简化绩效考核流程 |
| ratify | 汇报线逐级审定 / 组织架构逐级审定 |
| confirm | 员工确认流程 |
| appeal | 绩效申诉流程 |
| interview | 绩效面谈流程 |

| 周期类型 | select | string | 是 | 否 | — | 月度/半年度/年度/自定义 |
| 方案编号 | input text | string | — | 是 | — | 自动生成，禁用 |

## 9. 侧边栏底部 — 配置完成度

| 字段名 | 控件 | 计算逻辑 |
|--------|------|----------|
| 进度条 | bar | `(done / total) * 100` |
| 百分比 | 文本 | 格式"50%" |

### 完成度计算

```
total = enabled阶段数 + 2（基本信息+考核组）
done = status='done'的阶段数 + (form.name ? 1 : 0) + (form.groups.length > 0 ? 1 : 0)
pct = min(100, round(done/total * 100))
```

## 10. 提交校验规则

| 校验项 | 规则 | 错误提示 | 阻断 |
|--------|------|----------|------|
| 方案名称 | 必填 | "请填写方案名称" | 是（alert） |

### 校验时机

- 保存时调用 `submit()`
- 名称为空 → alert 提示 → 跳转到基本信息 → 重新渲染
- 名称非空 → alert"方案已提交：{名称}"

## 11. 页面交互逻辑汇总

### 11.1 基础操作

| 场景 | 行为 |
|------|------|
| 点击侧边栏目录项 | 切换 activeSection → 重新渲染内容区 |
| 点击环节概览项 | 跳转到该环节详情页 |
| 返回环节列表 | 点击"← 返回环节列表"按钮 |
| 切换配置模式 | 合并/按考核组 → 刷新当前环节配置 |
| 切换考核组 tab | 更新 activeGroupTab[stageId] → 刷新配置内容 |
| 切换周期类型 | 调用 applyTemplate → 自动启用/禁用阶段 → 刷新 |
| 切换等级规则 | 更新 form.gradeRule → 等级标签和强制结束评价选项同步更新 |
| 打开设置弹窗 | 点击齿轮按钮 → 弹窗显示阶段开关 |
| 切换阶段开关 | 实时更新 form.stages[i].enabled |
| 折叠/展开 Section | 点击标题栏 → toggle open 类 |
| 新增考核组 | 列表末尾添加默认考核组 |
| 删除考核组 | 直接删除 |
| 添加/删除规则 | 动态增删规则行 |
| 保存草稿 | 弹窗"方案已保存为草稿" |
| 提交 | 校验名称 → 通过则弹窗"方案已提交：{名称}" |

### 11.2 侧边栏完成度联动

| 操作 | 完成度更新 |
|------|-----------|
| 填写方案名称 | `basic` 目录项状态 → done（绿） |
| 添加考核组 | `groups` 目录项状态 → done（绿） |
| 配置考核环节 | 对应环节目录项状态 → done（绿）/ warn（橙）/ pending（灰） |
| 禁用阶段 | 对应环节从侧边栏隐藏 |

### 11.3 周期模板联动

| 周期类型 | 自动启用阶段 | 需要等级 |
|----------|-------------|----------|
| 月度 | goal, eval, ratify, confirm | 是 |
| 半年度 | interview | 否 |
| 年度 | interview, ratify | 是 |
| 自定义 | 无（全部手动配置） | 按需 |

### 11.4 操作反馈

| 操作 | 反馈方式 | 内容 |
|------|----------|------|
| 保存草稿 | 弹窗 | "方案已保存为草稿" |
| 提交成功 | 弹窗 | "方案已提交：{名称}" |
| 提交失败（名称为空） | 弹窗 → 跳转到基本信息 | "请填写方案名称" |
| 切换周期 | 无弹窗（实时生效） | 阶段列表自动更新 |
| 删除考核组 | 无确认（直接删除） | 列表立即更新 |
| 配置完成度不足 | 无阻断（允许保存草稿） | 完成度进度条显示当前百分比 |

### 11.5 异常态交互

| 异常场景 | UI 表现 | 用户操作 |
|----------|---------|----------|
| 网络断开保存 | 弹窗"保存失败" | 数据保留本地 |
| 网络断开提交 | 弹窗"提交失败" | 数据保留 |
| 会话超时 | 弹窗"会话已过期" → 跳转登录 | 重新登录后回到向导 |
| 方案名称为空提交 | 弹窗"请填写方案名称" → 自动跳转到基本信息 | 填写名称 |
| 等级规则未选择 | 强制结束评价下拉为空 | 先选择等级规则 |
| 强制分布参数未配置 | 审定环节状态显示 warn（橙） | 进入审定配置补充 |

### 11.6 Loading 态

| 场景 | 表现 |
|------|------|
| 页面初始化 | 内容区显示骨架屏 |
| 保存/提交中 | 按钮显示 spinner + 禁用 |
| 切换 section | 内容区短暂 loading |

# 员工确认表单 — 字段清单与交互规则

> 基于 `perf-form-confirmation.html` 原型提取，版本 v3 快照

---

## 1. 页面布局

### 1.1 两种模板

| 模板 | 布局 | 内容 |
|------|------|------|
| 月度考核 | 基本信息 → 考核目标与结果（层级表）→ 审定结果 → 确认操作 | 目标→考核→审定全流程 |
| 年度考核 | 基本信息 → 绩效面谈记录 → 审定结果 → 确认操作 | 面谈→审定，无目标考核表 |

### 1.2 整体结构

```
┌─────────────────────────────────────────────────────────┐
│ Header: 面包屑 + 流程状态 badge（待确认/已确认/已拒绝）  │
├─────────────────────────────────────────────────────────┤
│ Page Header: 标题 + 考核周期                            │
├─────────────────────────────────────────────────────────┤
│ 模板切换: 月度考核 | 年度考核                           │
├─────────────────────────────────────────────────────────┤
│ 月度模板:                                               │
│   ├─ 基本信息网格（3列）                                │
│   ├─ 考核目标与结果（层级表: 模块→目标）               │
│   │   ├─ 汇总卡片（加权总分 + 审定等级）               │
│   │   └─ 各环节总评语（按处理人角色分色）               │
│   └─ 审定结果（最终绩效等级）                           │
│                                                         │
│ 年度模板:                                               │
│   ├─ 基本信息网格（3列）                                │
│   ├─ 绩效面谈记录（面谈信息 + 面谈要点）               │
│   └─ 审定结果（最终绩效等级）                           │
├─────────────────────────────────────────────────────────┤
│ 确认操作: 提示文本 + [不同意] [确认无误] 按钮          │
├─────────────────────────────────────────────────────────┤
│ Status Display: 操作后显示确认/拒绝状态+时间            │
└─────────────────────────────────────────────────────────┘
```

## 6. 月度模板 — 考核目标与结果字段

### 6.1 层级表结构

```
模块行（蓝底高亮，colspan=7）
  ├─ 目标行: 模块(空)/目标名称/权重/衡量标准/完成值/完成进度/评价结果
  └─ ...
汇总区: 加权总分 + 审定等级
评语区: 各环节总评语（按角色分色）
```

### 6.2 表头

| 列名 | 宽度 | 说明 |
|------|------|------|
| 模块 | 100px | 模块分组行名 |
| 目标名称 | 140px | 指标名称 |
| 权重 | 60px | 百分比 |
| 衡量标准 | auto | 目标值/挑战值/保底值 + 评分规则 |
| 完成值/完成情况 | 110px | 定量=数值，定性=文字 |
| 完成进度 | 80px | 进度条 + 百分比 |
| 评价结果 | 80px | 等级标签 |

### 6.3 目标行字段

| 字段名 | 控件 | 数据源 | 只读 | 说明 |
|--------|------|--------|------|------|
| 模块 | 文本（分组行） | module | 是 | 蓝底高亮，带模块权重 |
| 目标名称 | 文本 | goal.name | 是 | |
| 权重 | 文本 | goal.weight | 是 | 格式"25%" |
| 衡量标准 | 文本 | goal.measure | 是 | 目标值/挑战值/保底值 + 评分规则描述 |
| 完成值 | 文本 | goal.completed | 是 | 定量=数值+单位，定性=文字 |
| 完成进度 | 进度条+文本 | 计算 | 是 | 进度条 + 百分比，颜色按进度变化 |
| 评价结果 | 等级标签 | goal.evaluation | 是 | 优秀(绿)/良好(蓝)/符合预期(灰)/低于预期(红) |

### 6.4 进度条颜色规则

```
progress >= 80% → green (--em)
progress >= 50% → blue (--pr)
progress >= 20% → orange (--ac)
progress < 20%  → gray (--c3)
```

### 6.5 等级标签颜色映射

| 等级 | CSS 类 | 背景色 | 文字色 |
|------|--------|--------|--------|
| 优秀 | `.excellent` | em50 | em700 |
| 良好 | `.good` | pr50 | pr |
| 符合预期 | `.normal` | c1 | c6 |
| 低于预期 | `.poor` | ro50 | ro |

### 6.6 汇总卡片字段

| 字段名 | 控件 | 数据源 | 说明 |
|--------|------|--------|------|
| 加权总分 | 文本 | 计算值 | 等宽字体，蓝色大号 |
| 审定等级 | 文本 | 审定结果 | 蓝色 |

### 6.7 各环节总评语字段

| 字段名 | 控件 | 数据源 | 说明 |
|--------|------|--------|------|
| 处理人角色 | 文本 | 评语来源 | 按角色分色显示 |
| 评语内容 | 文本 | 各环节评语 | 带左边框颜色标识 |

#### 评语角色颜色映射

| 角色 | 左边框色 | 标签色 |
|------|----------|--------|
| 员工自评 | pr（蓝） | pr |
| 直线上级 | em（绿） | em |
| 双线上级 | vi（紫） | vi |

## 11. 提交校验规则

### 确认无误

| 校验项 | 规则 | 处理 |
|--------|------|------|
| 无额外校验 | — | 直接提交确认 |
| 后处理 | 更新流程状态为"已确认" | 显示确认状态+时间 |

### 不同意

| 校验项 | 规则 | 处理 |
|--------|------|------|
| 无额外校验 | — | 直接提交异议 |
| 后处理 | 更新流程状态为"已拒绝" | 显示拒绝状态+时间，toast提示"已提交异议，等待 HRBP 处理" |

## 12. 页面交互逻辑汇总

### 12.1 基础操作

| 场景 | 行为 |
|------|------|
| 切换月度/年度模板 | 点击模板按钮 → 显示/隐藏对应模板区域 → 更新按钮选中状态 |
| 确认无误 | 直接 toast"确认成功" → 更新状态 badge 为"已确认" → 显示状态展示区 |
| 不同意 | toast"已提交异议，等待 HRBP 处理" → 更新状态 badge 为"已拒绝" → 显示状态展示区 |
| 页面初始化 | 渲染考核结果表格（月度模板） |

### 12.2 状态驱动 UI

| 状态 | UI 表现 |
|------|---------|
| `pending`（待确认） | 显示"确认无误"和"不同意"两个操作按钮；数据全部只读 |
| `confirmed`（已同意） | 操作按钮隐藏；显示绿色"已确认"状态标签 |
| `pending_hrbp_ack`（HRBP知悉中） | 操作按钮隐藏；显示橙色"异议处理中"状态标签 |
| `completed`（流程结束） | 操作按钮隐藏；显示绿色"已完成"状态标签 |

### 12.3 数据展示规则

| 数据区 | 来源 | 展示方式 |
|--------|------|----------|
| 基本信息 | 系统自动带出 | 只读文本 |
| 考核结果 | Evaluation 表单引用 | 只读表格（指标/完成值/评价/总分） |
| 审定结果 | Ratification 表单引用 | 只读文本（调整后等级） |
| 面谈结果 | Interview 表单直接引用 | 只读文本（面谈类型/亮点/不足/规划/需求） |
| 面谈附件 | Interview 表单直接引用 | 附件列表（可下载） |

### 12.4 操作反馈

| 操作 | 反馈方式 | 内容 |
|------|----------|------|
| 确认同意 | toast + 状态标签更新 | "确认成功" |
| 提交异议 | toast + 状态标签更新 | "已提交异议，等待 HRBP 处理" |
| HRBP 知悉 | 无前端操作（HRBP 后台处理） | — |
| 超时自动确认 | 通知 | "您已超时未操作，系统已自动确认为同意" |

### 12.5 跨页面联动

| 触发事件 | 联动行为 |
|----------|----------|
| 员工确认同意（`completed`） | 流程结束，无下游联动 |
| 员工提交异议（`pending_hrbp_ack`） | HRBP 收到待办通知 |
| HRBP 知悉（`completed`） | 流程结束，线下处理 |
| 超时自动确认（`completed`） | 流程结束，员工收到自动确认通知 |

### 12.6 异常态交互

| 异常场景 | UI 表现 | 用户操作 |
|----------|---------|----------|
| 网络断开确认 | toast 红色"操作失败" | 数据保留，可重试 |
| 会话超时 | 弹窗"会话已过期" → 跳转登录 | 重新登录后回到表单 |
| 面谈数据未完成（年度考核） | 面谈结果区域显示"面谈未完成"灰色提示 | 等待面谈完成 |
| 审定结果未完成 | 审定结果区域显示"审定未完成"灰色提示 | 等待审定完成 |

### 12.7 Loading 态

| 场景 | 表现 |
|------|------|
| 页面初始化 | 各数据区域显示骨架屏 |
| 确认/异议提交中 | 按钮显示 spinner + 禁用 |

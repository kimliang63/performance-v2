## 一、全局规则（所有流程共用）

### 1.1 角色定义

| 角色 | 来源 | 说明 |
|------|------|------|
| 被考核人 | 表单字段（流程实例关联） | 员工本人 |
| 直接上级 | 汇报关系 | 员工的直接汇报上级 |
| 间接上级 | 汇报关系 | 直接上级的直接上级 |
| 双线上级 | 花名册独立维护 | 员工的另一个上级（独立于汇报关系） |
| 逐级上级 | 汇报关系 / 组织架构 | 按审定模式决定来源 |
| HRBP | 组织角色 | 被考核人所在部门的HRBP |
| 停止人员 | 用户配置 | 汇报线审定模式下，到特定高管结束 |

### 1.2 空值处理规则

> **注意**：审批人为空时的跳过逻辑是**流程引擎已有标准能力**（节点参数配置"审批人为空时→自动跳过"），不需要业务功能逻辑额外处理。节点处理人的查询逻辑（汇报关系/组织角色/花名册）也是流程引擎标准能力。

| 场景 | 处理方式 | 流程引擎配置项 |
|------|---------|--------------|
| 间接上级为空 | 自动跳过 | 节点参数：审批人为空时→自动跳过 |
| 直接上级为空 | 自动跳过 | 节点参数：审批人为空时→自动跳过 |
| HRBP为空 | 自动跳过 | 节点参数：审批人为空时→自动跳过 |
| 逐级上级为空 | 自动跳过 | 节点参数：审批人为空时→自动跳过 |
| 双线上级为空 | 自动跳过 | 节点参数：审批人为空时→自动跳过 |

### 1.3 审批人数量规则

| 角色 | 人数 | 说明 |
|------|------|------|
| 间接上级 | **1人或为空** | 系统设计，不存在多人审批 |
| 直接上级 | **1人或为空** | 系统设计，不存在多人审批 |
| 逐级上级 | **逐级单人** | 每级1人，按汇报线/组织架构逐级流转 |
| HRBP | **1人或为空** | 单人审批 |
| 被考核人 | **1人** | 单人确认 |

## 二、流程流转逻辑

### 2.1 目标制定（2个流程）

**流程1：通用PBC目标制定** (`flow_goal_setting_pbc`)
```
员工填写 → 间接上级审批(可跳过) → 直接上级审批(可跳过) → 结束
```

**流程2：线上导入目标制定** (`flow_goal_setting_import`)
```
HRBP批量导入 → 员工确认 → 间接上级审批(可跳过) → 直接上级审批(可跳过) → 结束
```

| 逻辑点 | 通用PBC | 线上导入 |
|--------|---------|---------|
| 发起人 | 员工 | HRBP |
| 员工确认环节 | 无（员工自己填的） | 有（HRBP导入后员工确认） |
| 员工不同意 | / | 退回HRBP重新导入 |
| 间接/直接上级审批 | 同意/不同意/驳回到发起人/转交 | 同意/不同意/驳回到发起人/转交 |

**驳回流转**：
- 间接上级驳回 → 退回被考核人（发起人）
- 直接上级驳回 → 退回被考核人（发起人）
- 驳回后员工修改 → 重新进入审批流

---

### 2.2 绩效考核（1个流程）

**流程3：线上考核** (`flow_performance_review_online`)
```
被考核人自评 → 间接上级评分(可跳过) → 直接上级评分 → 结束
```

| 逻辑点 | 配置 |
|--------|------|
| 自评节点 | 员工填写完成值+评价结果+评语 |
| 间接上级评分 | 查看自评结果，选择各指标评价+评语 |
| 直接上级评分 | 查看自评+间接上级评分，选择各指标评价+总评价结果+总评语 |
| 总评价结果 | 根据绩效考核环节配置的**节点权重**加权求和计算；若未配节点权重，默认按最后一个填写的计算结果 |

**表单字段权限差异**：

| 字段 | 自评(节点1) | 间接上级(节点2) | 直接上级(节点3) |
|------|-----------|----------------|----------------|
| 完成值 | 可编辑/必填 | 只读 | 只读 |
| 评价结果(各指标) | 可编辑/必填 | 可编辑/必填 | 可编辑/必填 |
| 评语 | 可编辑 | 可编辑 | 可编辑 |
| 总评价结果 | 只读 | 只读 | **可编辑/必填** |
| 总评语 | 只读 | 只读 | **可编辑** |

---

### 2.3 结果审定（2个流程）

**流程4：汇报线逐级审定** (`flow_result_determination_reporting_line`)
```
直接上级评级 → 间接上级评级(可跳过) → 逐级上级评级 → HRBP审批(仅远超预期) → 结束
```

**流程5：组织架构逐级审定** (`flow_result_determination_org_structure`)
```
直接上级评级 → 间接上级评级(可跳过) → 逐级上级评级 → HRBP审批(仅远超预期) → 结束
```

| 差异项 | 汇报线逐级(04) | 组织架构逐级(05) |
|--------|---------------|-----------------|
| 逐级上级来源 | 汇报关系 | 组织架构 |
| 停止规则 | 到特定高管结束（用户配置） | 无停止规则（到顶层自动结束） |

> **V3依赖**：当前流程引擎不支持"逐级审批+特定规则停止"能力，需合并到V3流程引擎版本中实现。

**审定条件判断**：
1. 强制分布校验：弱控→提示可提交；强控→禁止提交
2. HRBP审批条件：调整后等级=A（远超预期）→ 进入HRBP审批；否则跳过
3. 停止人员判断（仅汇报线模式）：到k总/彭总/华辉总 → 结束审定
4. 跳过审定的员工 → 绩效方案中定义固定绩效结果等级

---

### 2.4 面谈（1个流程）

**流程7：绩效面谈** (`flow_interview`)
```
员工填写面谈记录 → 直接上级审批 → 结束
```

| 逻辑点 | 配置 |
|--------|------|
| 员工填写内容 | 面谈类型/亮点≥3项/不足≥3项/下阶段规划/需求≥1项/附件 |
| 直接上级审批 | 填写上级评语（必填），同意/不同意 |
| 不同意 | 驳回到员工重新填写 |

---

### 2.5 结果确认（1个流程）

**流程6：通用确认+申诉** (`flow_result_confirm`)
```
员工确认 → [同意] → 结束
          → [不同意] → HRBP知悉 → 结束
```

| 逻辑点 | 配置 |
|--------|------|
| 员工操作 | 查看考核结果+面谈结果+审定结果（全部只读），点击同意/不同意 |
| HRBP知悉 | 员工不同意时触发，查看后流程结束 |
| 不确认结果 | **线下处理**，HRBP不驳回员工 |

## 三、跨流程数据流转

### 3.1 环节触发顺序

**场景1：月度考核**
```
目标制定完成 → 绩效考核完成 → 结果审定完成 → 结果确认完成
```

**场景2：年度考核**
```
面谈完成 → 结果审定完成 → 结果确认完成
```

**确认**：确认永远在最后。月度考核无面谈环节，年度考核无目标制定和绩效考核环节。

### 3.2 表单数据级联

| 数据项 | 来源 | 流向 |
|--------|------|------|
| 基本信息（被考核人/部门/职位/周期） | 系统自动带出 | 所有表单 |
| 目标信息（目标名称/权重/衡量标准） | 目标制定表单 | → 考核表单（引用，只读） |
| 完成值 | 考核表单（员工填写） | → 审定表单（引用，只读） |
| 各指标评价 | 考核表单（上级填写） | → 审定表单（引用，只读） |
| 总评价结果 | 考核表单（直接上级计算） | → 审定表单（引用，只读） |
| 审定结果（调整后等级） | 审定表单 | → 确认表单（引用，只读） |
| 面谈结果 | 面谈表单 | → 确认表单（引用，只读） |
| 强制分布数据 | 活动实例化配置（继承自方案，可修改） | → 审定表单（实时展示） |

```
→ HRBP查看全部只读，点击"知悉"→ 流程结束
→ 不确认结果线下处理，HRBP不驳回员工
```

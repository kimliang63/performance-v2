# 流程5：组织架构逐级审定

## 一、流程概述

| 项目 | 内容 |
|------|------|
| 流程名称 | 组织架构逐级审定 |
| 流程编码 | flow_result_determination_org_structure |
| 流程说明 | 按组织架构逐级审定绩效等级 |
| 适用场景 | 组织架构审定模式（用户在绩效方案配置时勾选） |

### 流程节点

```
开始 → 直接上级评级 → 间接上级评级（可跳过）→ 逐级上级评级（可跳过）→ HRBP审批（仅远超预期）→ 结束
```

### 4.1 流程基本信息

| 流程编码 | flow_result_determination_org_structure |

| 审定模式 | 组织架构逐级（用户在绩效方案配置时勾选） |

### 4.2 节点处理人角色

| 角色编码 | 角色名称 | 角色来源 | 说明 |
|---------|---------|---------|------|
| ROLE_DIRECT_LEADER | 直接上级 | 汇报关系 | 被考核人的直接汇报上级 |
| ROLE_INDIRECT_LEADER | 间接上级 | 汇报关系 | 直接上级的直接上级 |
| ROLE_HIERARCHY_LEADER | 逐级上级 | 组织架构 | 组织架构逐级上级 |
| ROLE_HRBP | HRBP | 组织角色 | 被考核人部门HRBP |

### 4.3 与汇报线逐级审定的差异

| 差异项 | 汇报线逐级审定 | 组织架构逐级审定 |
|--------|--------------|----------------|
| 逐级上级来源 | 汇报关系 | 组织架构 |
| 停止规则 | 到特定高管结束（用户配置） | 无停止规则（到顶层自动结束） |
| 适用场景 | 有明确汇报线停止点 | 按组织架构自然结束 |

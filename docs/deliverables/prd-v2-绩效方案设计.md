## 一、需求概述

### 1.1 需求背景

1. V1 基础配置（绩效周期/单位/等级/计算规则/衡量标准/指标库/评价规则/强制分布）已完成评审并锁定，需要在配置层之上构建考核方案设计能力；
2. 绩效方案是考核活动的模板，定义考核组、考核环节、流程关联、表单关联、强制分布参数等，活动发起时 copy 方案配置；
3. 当前方案设计需要支持多考核组（按部门/职位/职级条件匹配）、多环节配置（目标制定/绩效考核/结果审定/结果确认/绩效申诉/绩效面谈）、以及与流程引擎的流程模型关联。

### 1.2 目标用户

| 角色名称 | 用户场景 | 用户故事 |
|----------|----------|----------|
| COE | 制定绩效考核方案，配置考核组和考核环节 | 我需要创建考核方案，定义哪些人参加哪个考核组，每个环节用什么流程和表单 |
| 系统管理员 | 维护绩效基础配置，辅助方案设计 | 我需要确保方案配置的规则与基础配置一致 |
| HRBP | 使用方案发起考核活动 | 我需要基于方案模板快速发起一次绩效考核活动 |

## 二、需求目标

1. 支持 COE 创建/编辑/复制/启用/停用考核方案
2. 支持多考核组配置，按条件（部门/职位/职级）+ 人员交集确定考核范围
3. 支持向导式配置：基本信息 → 考核组 → 考核环节设计
4. 各考核环节可独立配置开启/结束方式、流程模板、通知、强制分布参数
5. 方案停用后已发起的活动不受影响

#### 4.11 方案与活动关系

```
方案（V2配置）               活动（V4执行）
┌──────────────────┐       ┌──────────────────┐
│ 考核组            │ copy→│ 参与人员（可微调） │
│ 环节配置          │ copy→│ 环节配置（可修改） │
│ 流程模板          │ copy→│ 流程模板          │
│ 表单模板          │ copy→│ 表单模板          │
│ 强制分布          │ copy→│ 强制分布（可修改） │
│ 评价规则          │ copy→│ 评价规则          │
│ 节点权重          │ copy→│ 节点权重（可修改） │
└──────────────────┘       └──────────────────┘
                                 │
                                 ▼
                           最终按活动配置执行
```

1. 活动实例化时 copy 方案全部配置
2. 活动可修改，执行时只看活动配置
3. 方案停用后，已发起的活动不受影响

#### 4.12 关键设计决策

| 决策项 | 结论 |
|--------|------|
| 考核组模式 | 支持多组，按条件+人员交集确定范围 |
| 强制分布控制 | 强控（不允许提交）和弱控（提示后可提交） |
| 流程模板关联 | 关联流程引擎中的流程模型 |
| 绩效申诉 | V2 标记为待定 |
| 评价规则 | 在考核组配置中下拉选择，引用V1.7评价规则 |
| 审定模式/停止人员/HRBP知悉节点 | 均在流程配置中定义，方案只选择用哪个流程 |
| 共用环节配置 | 保留，多考核组可能共用部分环节 |
| 角色权限 | 由组织人事模块维护，绩效不单独维护 |
| 消息通知 | 后续版本统一做 |

# PBC 目标制定 — 字段清单与交互规则

> 基于 `perf-form-pbc.html` 原型提取，版本 v3 快照

---

## 1. 页面布局

```
┌─────────────────────────────────────────────────────────┐
│ Header: 面包屑 + 流程状态 badge                          │
├─────────────────────────────────────────────────────────┤
│ Page Header: 标题 + 操作按钮 + 员工信息 + 汇总统计       │
├─────────────────────────────────────────────────────────┤
│ Module Section × N（可折叠）                             │
│   ├─ 模块标题栏: 名称 + 合计权重 + 目标数 + 操作按钮     │
│   └─ Goal Row × N（可展开详情）                          │
│       ├─ 摘要行: 拖拽 + 名称 + 类型标签 + 权重 + 操作   │
│       └─ 详情区: 字段网格 + 等级标准表                   │
├─────────────────────────────────────────────────────────┤
│ Bonus Section: 加减分项（内联编辑）                      │
├─────────────────────────────────────────────────────────┤
│ Action Bar (fixed bottom): 总权重 + 保存草稿 + 提交      │
├─────────────────────────────────────────────────────────┤
│ Edit Sidebar (right slide): 目标编辑表单                 │
│ Indicator Picker Modal: 指标选择弹窗                     │
│ Copy Modal: 复制历史目标弹窗                             │
│ Language Modal: 多语言编辑弹窗                           │
└─────────────────────────────────────────────────────────┘
```

## 2. 数据结构

### 2.1 Goal 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识，格式 `g{timestamp}` |
| `name` | string | 目标名称 |
| `indicator` | string | 关联指标名称（从指标库带入） |
| `type` | `'quant' \| 'qual'` | 定量 / 定性 |
| `weight` | number | 权重百分比 |
| `desc` | string | 目标描述 |
| `measure` | string | 衡量方式（定量从指标带入，定性用户输入） |
| `measureText` | string | 定性目标的衡量标准文本 |
| `target` | number | 目标值（定量） |
| `challenge` | number | 挑战值（定量） |
| `baseline` | number | 保底值（定量） |
| `rule` | string | 计算规则名称（定量从指标带入） |
| `ruleFormula` | string | 计算公式文本（定量从指标带入） |
| `msLevels` | array | 等级标准数组，从指标带入 |
| `linkedIndicatorId` | string \| null | 关联的指标 ID |
| `i18n` | object | 多语言内容 `{name:{zh,en}, desc:{zh,en}}` |

### 2.2 Module 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 模块名称 |
| `weight` | number | 模块预设权重（仅展示，不参与计算） |
| `icon` | string | 图标标识 |
| `color` | string | 主题色 |
| `accent` | string | 样式类名 |
| `goals` | Goal[] | 该模块下的目标数组 |

### 2.3 BonusItem 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `reason` | string | 原因说明 |
| `type` | `'bonus' \| 'penalty'` | 加分 / 减分 |
| `score` | number | 分值（减分为负数） |

| 字段名 | 控件 | 类型 | 必填 | 校验 | 默认值 | 说明 |
|--------|------|------|------|------|--------|------|
| 原因说明 | input text | string | 否 | 0-200字符 | 空 | |
| 扣/加分 | select | enum | 否 | — | 'bonus' | 选项：加分/减分 |
| 分值 | input number | number | 否 | step=0.5, ≥0 | 0 | 显示时减分自动加负号 |

- 切换"加分/减分"时，自动调整 score 的正负号
- 分值输入框只接受非负数，type 为减分时内部存储为负数

## 12. 提交校验规则

| 校验项 | 规则 | 错误提示 | 阻断 |
|--------|------|----------|------|
| 总权重 | 所有目标权重之和必须等于 100% | "总权重 XX%，需等于 100%" | 是（toast 提示） |
| 目标名称 | 必填，1-100字符 | — | 是（保存时） |
| 权重 | 必填，0-100 整数 | — | 是（保存时） |
| 目标值（定量） | 必填，数字 >0 | — | 是（保存时） |
| 关联指标（定量） | 必须选择 | — | 是（保存时） |

### 校验时机

- **实时显示**：总权重不等于 100% 时，页面信息区和底部操作栏均显示红色警告
- **提交时**：调用 `validateWeights()` 检查，不通过则 toast 提示并阻止提交
- **保存草稿**：不做校验，直接保存

---

## 13. 总分计算逻辑

```
总分 = Σ(目标得分 × 目标权重/100) + 加减分合计
```

- **定量目标得分**：`min(target > 0 ? round(实际/目标 × 100) : 0, 120)`
- **定性目标得分**：`gradeToScore(grade)` 映射表
- **加减分**：直接累加（减分为负数）

> 注：PBC 阶段不填写完成值和评分，总分计算在考核阶段使用。此公式为考核阶段的计算逻辑。

## 14. 页面交互逻辑汇总

### 14.1 基础操作

| 场景 | 行为 |
|------|------|
| 新增定量目标 | 弹出类型选择 → 打开指标选择弹窗 → 选择后自动填充并打开编辑侧边栏 |
| 新增定性目标 | 弹出类型选择 → 直接打开空白编辑侧边栏 |
| 编辑目标 | 点击编辑按钮 → 侧边栏滑入，主内容右移 |
| 保存编辑 | 校验必填项 → 更新数据 → 关闭侧边栏 → 刷新列表 |
| 删除目标 | confirm 确认 → 删除 → 刷新 |
| 折叠/展开模块 | 点击标题栏箭头 → 动画展开/折叠 |
| 展开目标详情 | 点击摘要行 → 展开详情区（等级标准表等） |
| 切换考核组 tab | 刷新当前配置内容 |
| 拖拽排序 | 目标行左侧拖拽图标（原型仅 UI，未实现排序逻辑） |

### 14.2 状态驱动 UI

| 状态 | UI 表现 |
|------|---------|
| `goal_setting`（员工填写中） | 所有字段可编辑；显示"保存草稿"和"提交"按钮 |
| `pending_indirect_approval` | 所有字段只读；顶部显示"审批中"状态条；审批人看到"同意/驳回"按钮 |
| `pending_direct_approval` | 所有字段只读；直接上级看到"同意/驳回"按钮 |
| `approved` | 所有字段只读；显示绿色"已通过"状态标签 |
| `rejected` | 所有字段可编辑（允许修改后重新提交）；显示红色"已驳回"状态标签 + 驳回原因 |

### 14.3 权限态交互

| 角色 | 可见范围 | 可编辑字段 |
|------|----------|-----------|
| 被考核人 | 自己的目标表单 | 全部字段（填写阶段） |
| 间接上级 | 被考核人目标（只读） | 无（仅审批操作） |
| 直接上级 | 被考核人目标（只读） | 无（仅审批操作） |
| HRBP | 所有导入员工的目标 | 导入时可编辑 |

### 14.4 操作反馈

| 操作 | 反馈方式 | 内容 |
|------|----------|------|
| 保存草稿 | toast（底部，3秒消失） | "草稿已保存" |
| 提交成功 | toast + 页面状态变更 | "目标已提交，等待间接上级审批" |
| 提交失败（校验不通过） | toast（红色） | 具体校验错误信息 |
| 提交失败（网络异常） | 弹窗 | "网络异常，提交失败，请稍后重试" |
| 审批通过 | toast + 状态标签更新 | "审批已通过" |
| 审批驳回 | toast + 状态标签更新 | "已驳回，原因：{comment}" |
| 目标保存中 | 按钮 loading 态 | 按钮显示 spinner，禁用重复点击 |

### 14.5 跨页面联动

| 触发事件 | 联动行为 |
|----------|----------|
| 目标审批通过（`approved`） | 考核表单自动创建，目标数据写入考核表单只读区 |
| 目标被驳回（`rejected`） | 员工收到通知，可修改后重新提交 |
| HRBP 批量导入完成 | 员工收到确认通知，进入员工确认环节 |
| 员工确认导入目标不同意 | 退回 HRBP，HRBP 收到退回通知 |

### 14.6 异常态交互

| 异常场景 | UI 表现 | 用户操作 |
|----------|---------|----------|
| 网络断开保存 | toast 红色"保存失败" | 数据保留在本地，网络恢复后可重试 |
| 网络断开提交 | toast 红色"提交失败" | 数据保留，不触发流程流转 |
| 会话超时 | 弹窗"会话已过期" → 跳转登录 | 重新登录后回到表单，草稿已保存 |
| 并发驳回（员工编辑中被驳回） | 弹窗"目标已被驳回，请查看最新状态" → 强制刷新 | 员工查看驳回原因后修改 |
| 目标数量 < 2 | toast 红色"至少需要 2 个目标" | 继续编辑 |
| 总权重 ≠ 100% | 底部操作栏红色警告 + 提交时 toast | 调整权重 |

### 14.7 Loading 态

| 场景 | 表现 |
|------|------|
| 页面初始化 | 表单区域显示骨架屏（Skeleton） |
| 保存/提交中 | 按钮显示 spinner + 禁用 |
| 指标弹窗加载 | 弹窗内显示 loading spinner |
| 审批操作中 | 审批按钮显示 spinner + 禁用 |

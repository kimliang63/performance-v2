# 实时更新 vs 版本管理 信息分类

> 版本 v1 | 2026-06-18

---

## 一、信息分类总览

| 分类 | 说明 | 更新方式 | 影响范围 |
|------|------|----------|----------|
| **实时更新** | 修改后立即生效，所有引用处同步更新 | 自动生效 | 全局/方案级 |
| **版本管理** | 修改后需手动更新，已有引用保持原值 | 手动更新/快照 | 仅影响新引用 |
| **快照不可变** | 生成后不可修改，作为历史记录 | 不可变 | 仅影响当前实例 |

## 二、实时更新的信息

### 2.1 基础配置层（V1）

| 配置项 | 实时更新 | 说明 |
|--------|----------|------|
| 绩效周期 | ✅ 是 | 修改后所有方案读取最新周期配置 |
| 单位 | ✅ 是 | 修改后所有方案读取最新单位配置 |
| 等级规则 | ✅ 是 | 修改后未启用方案读取最新，已启用方案新活动读取最新 |
| 计算规则 | ✅ 是 | 修改后未启用方案读取最新，已启用方案新活动读取最新 |
| 衡量标准 | ✅ 是 | 修改后未启用方案读取最新，已启用方案新活动读取最新 |
| 指标库 | ✅ 是 | 修改后未启用方案读取最新，已启用方案新活动读取最新 |
| 评价规则 | ✅ 是 | 修改后未启用方案读取最新，已启用方案新活动读取最新 |
| 强制分布规则 | ✅ 是 | 修改后未启用方案读取最新，已启用方案新活动读取最新 |

**实时更新机制**：
```yaml
查询方案配置时:
  1. 检查方案状态
  2. 若为草稿(draft): 直接读取基础配置最新值
  3. 若为已启用(enabled): 
     - 草稿方案读取最新值
     - 已启用方案的新活动启动时读取最新值
  4. 若为已停用(disabled): 不读取基础配置
```

### 2.2 方案配置层（V2）- 草稿状态

| 配置项 | 实时更新 | 说明 |
|--------|----------|------|
| 方案名称 | ✅ 是 | 草稿方案可随时修改 |
| 考核组配置 | ✅ 是 | 草稿方案可随时修改 |
| 考核环节配置 | ✅ 是 | 草稿方案可随时修改 |
| 流程模板 | ✅ 是 | 草稿方案可随时修改 |
| 表单模板 | ✅ 是 | 草稿方案可随时修改 |
| 强制分布配置 | ✅ 是 | 草稿方案可随时修改 |

**实时更新机制**：
```yaml
草稿方案修改:
  1. COE修改方案配置
  2. 保存时版本号自动+1
  3. 所有引用该方案的地方读取最新配置
  4. 无进行中活动影响
```

### 2.3 活动配置层（V4）- 活动级覆盖

| 配置项 | 实时更新 | 说明 |
|--------|----------|------|
| 通知配置 | ✅ 是 | 活动进行中可随时修改 |
| 环节开启/结束方式 | ✅ 是 | 未开始的环节可修改 |
| 环节时间范围 | ✅ 是 | 未开始的环节可修改 |

**实时更新机制**：
```yaml
活动配置修改:
  1. HRBP修改活动级覆盖配置
  2. 保存后立即生效
  3. 未开始的环节使用新配置
  4. 已开始的环节保持原配置
```

## 三、版本管理的信息

### 3.1 基础配置层（V1）- 被方案引用时

| 配置项 | 版本管理 | 说明 |
|--------|----------|------|
| 等级规则 | ⚠️ 是 | 已启用方案引用时，读取快照中的值 |
| 计算规则 | ⚠️ 是 | 已启用方案引用时，读取快照中的值 |
| 衡量标准 | ⚠️ 是 | 已启用方案引用时，读取快照中的值 |
| 指标库 | ⚠️ 是 | 已启用方案引用时，读取快照中的值 |
| 评价规则 | ⚠️ 是 | 已启用方案引用时，读取快照中的值 |
| 强制分布规则 | ⚠️ 是 | 已启用方案引用时，读取快照中的值 |

**版本管理机制**：
```yaml
已启用方案引用基础配置:
  1. 活动启动时生成 scheme_snapshot
  2. 快照中包含基础配置的当前值
  3. 活动执行时读取快照中的值
  4. 基础配置更新不影响已进行的活动
  5. 新活动启动时读取基础配置最新值
```

### 3.2 方案配置层（V2）- 已启用状态

| 配置项 | 版本管理 | 说明 |
|--------|----------|------|
| 考核组配置 | ⚠️ 是 | 已启用方案修改后，新活动使用新配置 |
| 考核环节配置 | ⚠️ 是 | 已启用方案修改后，新活动使用新配置 |
| 流程模板 | ⚠️ 是 | 已启用方案修改后，新活动使用新配置 |
| 表单模板 | ⚠️ 是 | 已启用方案修改后，新活动使用新配置 |
| 强制分布配置 | ⚠️ 是 | 已启用方案修改后，新活动使用新配置 |

**版本管理机制**：
```yaml
已启用方案配置更新:
  1. COE修改已启用方案配置
  2. 保存时版本号自动+1
  3. 已进行的活动使用原快照配置
  4. 新活动启动时读取方案最新配置
  5. 生成新的 scheme_snapshot
```

### 3.3 活动配置层（V4）- 快照生成后

| 配置项 | 版本管理 | 说明 |
|--------|----------|------|
| 方案快照 | ⚠️ 是 | 活动启动时生成，执行中不可变 |
| 参与者列表 | ⚠️ 是 | 活动启动时生成，动态考核除外 |
| 环节状态 | ⚠️ 是 | 活动启动时初始化，按流程推进 |

**版本管理机制**：
```yaml
活动快照管理:
  1. 活动启动时从方案 copy 全部配置
  2. 生成 scheme_snapshot（JSON）
  3. 活动执行中读取快照配置
  4. 方案配置更新不影响已进行的活动
  5. 快照不可修改
```

## 四、快照不可变的信息

### 4.1 活动快照（scheme_snapshot）

| 字段 | 不可变 | 说明 |
|------|--------|------|
| 方案基本信息 | ✅ 是 | 活动启动时的方案配置 |
| 考核组配置 | ✅ 是 | 活动启动时的考核组配置 |
| 考核环节配置 | ✅ 是 | 活动启动时的环节配置 |
| 流程模板 | ✅ 是 | 活动启动时的流程模板 |
| 表单模板 | ✅ 是 | 活动启动时的表单模板 |
| 强制分布配置 | ✅ 是 | 活动启动时的强制分布配置 |
| 基础配置快照 | ✅ 是 | 活动启动时的基础配置值 |

**快照机制**：
```yaml
快照生成:
  时机: 活动启动时（start操作）
  内容: 方案全部配置 + 基础配置快照
  存储: perf_activity.scheme_snapshot 字段
  修改: 不可修改（数据库层面禁止更新）

快照使用:
  场景1: 活动执行时读取配置
  场景2: 活动监控时展示配置
  场景3: 活动导出时包含配置
```

### 4.2 历史数据（明细表）

| 字段 | 不可变 | 说明 |
|------|--------|------|
| goal_detail | ✅ 是 | 目标制定历史记录 |
| eval_detail | ✅ 是 | 绩效考核历史记录 |
| ratification_detail | ✅ 是 | 结果审定历史记录 |
| interview_detail | ✅ 是 | 绩效面谈历史记录 |
| confirmation | ✅ 是 | 结果确认记录 |

**历史数据机制**：
```yaml
追加式存储:
  模式: 每次submit新增一行记录
  状态: active（当前有效）/ voided（历史作废）
  查询: 取active最新行 = 当前数据
  历史: voided行保留 = 历史可追溯

数据不可变:
  已提交的数据不可修改
  驳回重提时新增行，旧行标记为voided
  终态数据（approved/completed/confirmed）不可回退
```

## 五、配置更新决策树

### 5.1 基础配置更新决策

```
COE 修改基础配置
      ↓
检查是否有方案引用该配置
      ↓
  ┌─ 无方案引用 → 直接更新，立即生效
  │
  └─ 有方案引用 → 检查方案状态
                    ↓
              ┌─ 草稿方案 → 直接更新，立即生效
              │
              └─ 已启用方案 → 检查是否有进行中活动
                              ↓
                        ┌─ 无进行中活动 → 直接更新，立即生效
                        │
                        └─ 有进行中活动 → 提示"不影响已进行活动，是否继续？"
                                          ↓ 确认
                                    更新配置
                                          ↓
                                    新活动启动时读取最新配置
```

### 5.2 方案配置更新决策

```
COE 修改方案配置
      ↓
检查方案状态
      ↓
  ┌─ 草稿方案 → 直接更新，立即生效
  │
  └─ 已启用方案 → 检查是否有进行中活动
                    ↓
              ┌─ 无进行中活动 → 直接更新，立即生效
              │
              └─ 有进行中活动 → 提示"不影响已进行活动，是否继续？"
                                ↓ 确认
                          更新配置
                                ↓
                          新活动启动时读取最新配置
```

### 5.3 活动配置更新决策

```
HRBP 修改活动配置
      ↓
检查活动状态
      ↓
  ┌─ 草稿活动 → 直接更新，立即生效
  │
  └─ 进行中活动 → 检查修改的配置项
                    ↓
              ┌─ 通知配置 → 直接更新，立即生效
              │
              ├─ 未开始环节的配置 → 直接更新，立即生效
              │
              └─ 已开始环节的配置 → 不可修改
```

## 七、示例场景

### 场景1：等级规则更新

```
当前状态:
  - 方案A（草稿）引用等级规则X
  - 方案B（已启用）引用等级规则X
  - 活动C（进行中）使用等级规则X的快照

COE修改等级规则X为Y:
  1. 方案A（草稿）: 立即读取等级规则Y
  2. 方案B（已启用）: 
     - 无进行中活动: 立即读取等级规则Y
     - 有进行中活动: 新活动读取Y，已进行活动使用X快照
  3. 活动C（进行中）: 继续使用等级规则X快照
```

### 场景2：方案考核组更新

```
当前状态:
  - 方案A（已启用）配置考核组G1
  - 活动B（进行中）使用考核组G1快照

COE修改方案A的考核组为G2:
  1. 方案A（已启用）: 考核组变为G2
  2. 活动B（进行中）: 继续使用考核组G1快照
  3. 新活动C: 使用考核组G2
```

### 场景3：活动通知配置更新

```
当前状态:
  - 活动A（进行中）通知配置为站内信+飞书

HRBP修改活动A的通知配置为站内信+飞书+邮件:
  1. 活动A（进行中）: 立即生效，后续通知使用新配置
  2. 已发送的通知: 不受影响
```

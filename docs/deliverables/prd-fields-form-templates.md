## 2. 数据结构

### FormRecord 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 表单名称（唯一标识） |
| `cat` | string | 业务分类：`绩效` / `关键指标` |
| `type` | string | 类型：`系统预置` / `自定义` |
| `ver` | string | 版本号（如 v2.1） |
| `desc` | string | 表单描述 |
| `modifier` | string | 最近修改人 |
| `time` | string | 最近修改时间（YYYY-MM-DD） |

### FORM_STRUCTURE 对象

按表单名称索引，存储表单的字段结构定义：

```javascript
FORM_STRUCTURE = {
  '表单名称': {
    sections: [
      {
        name: '分区名称',       // 如：基本信息、目标设定
        fields: [
          {
            label: '字段标签',   // 如：考核周期
            type: '控件类型',    // 如：选择、文本、数字(%)
            required: boolean   // 是否必填
          }
        ]
      }
    ]
  }
}
```

#### 控件类型枚举

| 类型值 | 说明 |
|--------|------|
| 文本 | 单行文本输入 |
| 多行文本 | 多行文本输入 |
| 数字 | 数字输入 |
| 数字(%) | 百分比数字输入 |
| 数字(自动) | 自动计算，不可编辑 |
| 选择 | 下拉选择 |
| 选择(只读) | 下拉选择，只读展示 |
| 人员选择 | 人员选择器 |
| 日期 | 日期选择器 |
| 金额 | 金额输入 |
| 文本(只读) | 单行文本，只读展示 |

### VERSION_HISTORY 对象

按表单名称索引，存储版本变更记录：

```javascript
VERSION_HISTORY = {
  '表单名称': [
    {
      ver: 'v2.1',          // 版本号
      date: '2026-05-01',   // 变更日期
      desc: '变更说明'       // 变更描述
    }
  ]
}
```

## 4. 筛选栏字段

| 控件 | 类型 | 选项 | 说明 |
|------|------|------|------|
| 业务分类 | select | 全部业务分类 / 绩效 / 关键指标 | 按业务分类筛选 |
| 类型 | select | 全部类型 / 系统预置 / 自定义 | 按类型筛选 |
| 搜索框 | input text | — | 按表单名称模糊搜索 |
| 结果计数 | 文本 | — | 显示"共 N 项" |

### 筛选逻辑

- 三个条件为 AND 关系，同时生效
- 搜索为 `includes` 模糊匹配（不区分大小写）
- 任一条件变更即时触发 `applyFilters()`
- 无重置按钮，切换下拉选项至"全部"即可清空对应条件

## 7. 预览弹窗字段

### 7.1 弹窗结构

| 区域 | 内容 |
|------|------|
| 标题 | "预览 - {表单名称}" |
| 信息栏 | 类型标签 + 版本标签 + 业务分类 + 描述 |
| 内容区 | 按 sections 渲染表单结构 |
| 底部 | [关闭] 按钮 |

### 7.2 表单结构渲染

每个 section 渲染为：
```
┌─ section-hd ──────────────────────────┐
│ 🔲 {section.name}                     │
├────────────────────────────────────────┤
│ {field.label} [必填*]     {field.type} │  ← 每个 field 一行
│ {field.label}            {field.type}  │
└────────────────────────────────────────┘
```

| 字段 | 数据源 | 说明 |
|------|--------|------|
| section 标题 | section.name | 蓝色图标 + 加粗 |
| 字段标签 | field.label | 左侧，100px 固定宽 |
| 必填标记 | field.required | 红色 `*`，仅 required=true 显示 |
| 控件类型 | field.type | 右侧，蓝底标签 |

### 7.3 无结构数据

若 `FORM_STRUCTURE[form.name]` 不存在或 sections 为空，显示居中提示："表单结构数据暂未配置"。

## 8. 复制弹窗字段

### 8.1 弹窗结构

| 区域 | 内容 |
|------|------|
| 标题 | "复制表单" |
| 表单体 | 源表单（只读） + 新表单名称（必填） |
| 提示条 | 橙色左边框提示："复制后将生成自定义类型的新表单，可自由编辑。" |
| 底部 | [取消] [确认复制] |

### 8.2 字段

| 字段名 | 控件 | 数据源 | 说明 |
|--------|------|--------|------|
| 源表单 | input(readonly) | copyTarget.name | 灰色背景，不可编辑 |
| 新表单名称 | input text | — | 必填，默认值"{原名称}(副本)" |

### 8.3 确认复制逻辑

1. 校验名称非空，空值时输入框边框变红
2. 创建新 FormRecord，`type` 设为 `自定义`，`ver` 设为 `v1.0`
3. 深拷贝源表单的 `FORM_STRUCTURE` 到新表单
4. 创建初始版本记录到 `VERSION_HISTORY`
5. 关闭弹窗 → 重新筛选渲染

## 10. 新建表单弹窗字段

### 10.1 弹窗结构

| 区域 | 内容 |
|------|------|
| 标题 | "新建表单" |
| 表单体 | 表单名称（必填） + 业务分类（必填） + 描述（选填） |
| 底部 | [取消] [确认创建] |

### 10.2 字段

| 字段名 | 控件 | 类型 | 必填 | 说明 |
|--------|------|------|------|------|
| 表单名称 | input text | string | 是 | 自动聚焦 |
| 业务分类 | select | string | 是 | 默认"绩效"，可选"绩效"/"关键指标" |
| 描述 | input text | string | 否 | placeholder"简要描述表单用途" |

### 10.3 确认创建逻辑

1. 校验名称非空，空值时输入框边框变红
2. 创建新 FormRecord，`type` 设为 `自定义`，`ver` 设为 `v1.0`
3. 创建空的 `FORM_STRUCTURE`（`{sections:[]}`）
4. 创建初始版本记录到 `VERSION_HISTORY`
5. 关闭弹窗 → 重新筛选渲染

## 11. 页面交互逻辑汇总

### 11.1 基础操作

| 场景 | 行为 |
|------|------|
| 点击表单名称 | 打开预览弹窗 |
| 点击预览按钮 | 打开预览弹窗 |
| 点击编辑按钮（自定义） | 跳转到表单编辑页 |
| 点击编辑按钮（系统预置） | 禁用，tooltip 提示不可编辑 |
| 点击复制按钮 | 打开复制弹窗 |
| 点击版本历史按钮 | 打开版本历史弹窗 |
| 点击新建表单 | 打开新建表单弹窗 |
| 切换业务分类筛选 | 即时过滤 |
| 切换类型筛选 | 即时过滤 |
| 输入搜索关键词 | 即时过滤 |

### 11.2 操作反馈

| 操作 | 反馈方式 | 内容 |
|------|----------|------|
| 复制成功 | 弹窗关闭 + 列表刷新 | 新表单出现在列表中 |
| 新建成功 | 弹窗关闭 + 列表刷新 | 新表单出现在列表中 |
| 复制名称为空 | 输入框边框变红 | 无 toast |
| 新建名称为空 | 输入框边框变红 | 无 toast |
| 筛选 | 表格即时更新 | 无额外反馈 |

### 11.3 异常态交互

| 异常场景 | UI 表现 | 用户操作 |
|----------|---------|----------|
| 筛选结果为空 | 表格居中显示文件图标 + "暂无匹配的表单" | 修改筛选条件 |
| 表单结构数据缺失 | 预览弹窗显示"表单结构数据暂未配置" | 无 |

## 12. 与方案向导表单环节的关系

本页面管理的是**预置表单模板**，与方案向导中的表单选择环节关联：

| 关联点 | 说明 |
|--------|------|
| 方案向导 → 表单模板选择 | 从本页面的表单清单中选择预置表单 |
| 系统预置表单 | 所有方案共享，不可修改结构 |
| 自定义表单 | 管理员创建，可按部门/业务定制 |
| 表单结构 → 表单渲染 | FORM_STRUCTURE 定义了表单在实际考核流程中的字段布局 |

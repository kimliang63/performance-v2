# 绩效管理系统 · 功能清单与逻辑确认

## 一、数据模型总览

```
绩效方案模板（Plan Template）
├── 基本信息
├── 绩效等级定义（全方案统一）
├── 考核组 1..N（每组独立的人员规则）
│   ├── 目标制定（流程模型 + 配置）
│   ├── 绩效考核（流程模型 + 配置）
│   ├── 结果审定（流程模型 + 配置）
│   ├── 结果确认（流程模型 + 配置）
│   └── 绩效面谈（流程模型 + 配置）
└── 启用/禁用各环节

考核活动实例（Activity Instance）
├── 关联方案模板
├── 考核周期（起止时间）
├── 发布状态（草稿 → 已发布 → 进行中 → 已结束）
└── 各考核组 × 各环节 → 生成实际任务单
```

**关键约束**：
- 一个方案下，**绩效等级只有一套**，在基本信息中定义，所有考核组、所有环节共享
- 流程模型 = 流程 + 表单，不可解耦，选择模型即同时绑定两者
- 每个考核环节按考核组并行配置，不同组可选不同流程模型

## 二、基本信息配置项

| 字段 | 类型 | 说明 |
|------|------|------|
| 方案名称 | 文本 | 必填 |
| 所属部门 | 组织树选择 | 必填 |
| 方案编号 | 文本 | 自动生成，可修改 |
| 是否启用 | 开关 | 默认启用 |
| 向下公开 | 开关 | 下级部门是否可见 |
| 规则描述 | 多行文本 | 选填 |
| **绩效结果等级** | **等级列表配置** | **全方案统一的等级体系** |

### 绩效等级定义

| 字段 | 说明 |
|------|------|
| 等级名称 | 如：远超预期 / 符合预期+ / 符合预期 / 符合预期- / 低于预期 |
| 等级编码 | 系统标识 |
| 对应分数区间 | 选填，如 90-100 / 80-89 / ... |
| 排序 | 从高到低排列 |

默认提供 5 级模板，支持增删改。

## 三、考核组配置

| 字段 | 类型 | 说明 |
|------|------|------|
| 考核组名称 | 文本 | 如「特殊考核组」「默认考核组」 |
| 规则列表 | 多条规则（AND 关系） | 每条规则：字段 + 运算符 + 值 |
| 排除规则 | 选填 | 从匹配结果中排除特定人员 |

**规则字段可选**：部门、职位、职级、人员（指定个体）、入职时间、用工类型等

**运算符**：等于、不等于、包含、属于（多选）

**默认组**：系统预置一个「默认考核组」，规则为「所有人员」，不可删除（可修改规则）。

## 四、考核环节配置（每环节 × 每考核组独立配置）

### 4.1 目标制定

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 流程模型 | 下拉选择 | 合并的流程+表单模型，选择即绑定 |
| 配置流程模型 | 按钮 | 跳转流程引擎新建/编辑模型 |
| 预览 | 弹窗 | 查看流程图 + 表单详情 |
| 开启方式 | 单选 | 手动开启 / 跟随活动发布自动开启 |
| 结束方式 | 单选 | 手动结束 / 到达截止时间自动结束 |
| 开始时间 | 日期时间 | 条件显示 |
| 结束时间 | 日期时间 | 条件显示 |
| 强制结束处理规则 | 单选 | 不处理 / 标记为已提交 |
| 强制结束评价结果 | 单选 | 不填入 / 由上一节点填入 |

**流程模型选项示例**：
- 通用目标制定流程：员工本人填写 → 直接上级审批
- 简化目标制定流程：员工本人填写

### 4.2 绩效考核

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 流程模型 | 下拉选择 | 同上 |
| 预览 | 弹窗 | 查看流程图 + 表单详情 |
| **评分权重分配** | **开关 + 编辑** | **默认关闭；开启后显示可编辑权重表，默认 0，合计须 100%** |
| 开启/结束方式 | 同目标制定 | |
| 强制结束规则 | 同目标制定 | |
| 表单模块 | 自动推导 | 由流程模型绑定，展示模块及权重 |

**流程模型选项示例**：
- 通用绩效考核流程：员工自评 → 直线上级评分 → 区域BP审批
- 简化绩效考核流程：直线上级评分

**评分权重逻辑**：
- 每个流程节点可配权重，默认 0
- 审批类型节点标记「不参与评分」
- 权重合计必须等于 100%

## 五、侧边栏导航逻辑

| 节点 | 状态流转 |
|------|----------|
| 基本信息 | 始终可编辑，完成保存后标记「已完成」 |
| 考核组 | 基本信息完成后可编辑，至少一个组标记「已完成」 |
| 目标制定 ~ 绩效面谈 | 前置节点完成后解锁；启用=可配置，禁用=跳过 |

**状态类型**：
- `done`（已完成）：绿色圆点
- `active`（当前编辑）：蓝色圆点 + 蓝色边框
- `pending`（未开始/待解锁）：灰色圆点
- `skipped`（已跳过/已禁用）：灰色虚线圆点

**进度统计**：已完成节点数 / (总节点数 - 跳过节点数) × 100%，以环形进度条展示在侧边栏顶部。

## 六、业务流转逻辑（活动实例维度）

### 6.1 活动生命周期

```
创建活动（选择方案 + 考核周期）
  → 草稿态（可编辑）
  → 发布（按方案配置生成各组 × 各环节任务单）
  → 进行中（各环节按配置的开启/结束方式自动或手动推进）
  → 已结束（所有环节完成或强制结束）
```

### 6.2 单环节流转（以「绩效考核」为例）

```
活动发布 / 手动开启
  → 系统按考核组规则匹配人员，为每人生成任务单
  → 按流程模型的节点顺序逐节点流转：
      节点1: 员工自评 → 员工填写表单 → 提交
      节点2: 直线上级评分 → 上级收到通知 → 填写评分 → 提交
      节点3: 区域BP审批 → BP收到通知 → 审批 → 提交
  → 环节结束
  → 下一环节自动/手动开启
```

### 6.3 评分与等级计算

- **目标制定**：不涉及评分，用于审批目标内容
- **绩效考核**：核心评分环节，员工自评和上级评分按权重加权（权重可配，默认 0，合计 100%）
- **结果审定**：多级审定人直接给出绩效等级，不走分数加权
- **结果确认**：员工确认最终结果
- **绩效面谈**：双向填写面谈表
- 最终得分 → 对照基本信息中定义的绩效等级分数区间 → 自动映射等级

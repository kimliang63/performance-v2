# 配置与活动逻辑规格

**创建日期**：2026-06-18
**文档位置**：docs/deliverables/

---

## 核心概念

### 三层配置架构
```
V1 基础配置（全局共享）→ V2 方案配置（模板）→ V4 活动层（实例）
```

### 配置更新机制

| 分类 | 说明 | 更新方式 |
|------|------|----------|
| **实时更新** | 修改后立即生效 | 自动生效 |
| **版本管理** | 修改后需手动更新 | 手动更新/快照 |
| **快照不可变** | 生成后不可修改 | 不可变 |

---

## 关键设计决策

### 1. 快照机制
- **时机**：活动启动时从方案 copy 全部配置
- **存储**：`perf_activity.scheme_snapshot`（JSON）
- **使用**：活动执行中读取快照，不读取方案最新配置
- **不可变**：快照生成后不可修改

### 2. 版本管理
- **方案版本**：每次保存自动+1
- **版本历史**：存储在 `perf_scheme_version` 表
- **快照引用**：活动记录 `scheme_version` 字段

### 3. 配置覆盖
- **活动级覆盖**：存储在 `perf_activity_config_override` 表
- **优先级**：活动级覆盖 > 方案快照配置

---

## 实时更新的信息

| 层级 | 配置项 | 说明 |
|------|--------|------|
| V1 基础配置 | 绩效周期、单位 | 修改后立即生效 |
| V2 方案（草稿） | 全部配置 | 修改后立即生效 |
| V4 活动 | 通知配置、未开始环节配置 | 修改后立即生效 |

---

## 版本管理的信息

| 层级 | 配置项 | 说明 |
|------|--------|------|
| V1 基础配置被引用时 | 等级规则、计算规则等 | 已启用方案读取快照值 |
| V2 方案（已启用） | 考核组、环节配置等 | 新活动使用新配置 |
| V4 活动快照 | 全部配置 | 生成后不可变 |

---

## 关键业务规则

| 规则ID | 规则名称 |
|--------|----------|
| BR-100 | 基础配置更新不影响已发起活动 |
| BR-120 | 活动启动时生成快照 |
| BR-122 | 活动使用快照配置执行 |
| BR-124 | 快照不可变 |
| BR-130 | 草稿方案可自由修改 |
| BR-131 | 已启用方案修改影响新活动 |
| BR-132 | 已启用方案修改不影响进行中活动 |
| BR-150 | 活动继承方案全部配置 |
| BR-152 | 活动级配置覆盖优先 |

## 数据库变更

### 新增表
1. `perf_scheme_version` - 方案版本历史表
2. `perf_activity_config_override` - 活动配置覆盖表

### 变更字段
1. `perf_scheme.version` - 版本号
2. `perf_activity.scheme_version` - 方案版本号
3. `perf_activity.scheme_snapshot` - 方案配置快照

# 产品架构

## 项目背景
三合一项目合并 EHR、OEHR 绩效业务，统一在 GHR 门户进行全球绩效业务管理。
- PBU 迁出，不购买蓝凌 OA
- 集团和 GOFO 招聘系统需重构
- 绩效流程实现依赖新流程引擎平台


## 三层配置架构

```
┌─────────────────────────────────────────────────────────────────┐
│  V1 基础配置层（全局共享，COE维护）                                │
│  绩效周期 / 单位 / 等级规则 / 计算规则 / 衡量标准                  │
│  指标库 / 评价规则 / 强制分布规则                                  │
└─────────────────────────────────────────────────────────────────┘
                          ↓ FK 引用（只读）
┌─────────────────────────────────────────────────────────────────┐
│  V2 方案配置层（模板，COE维护）                                    │
│  绩效方案 → 考核组 / 考核环节 / 流程模板 / 表单模板                 │
└─────────────────────────────────────────────────────────────────┘
                          ↓ 快照 copy（不可变）
┌─────────────────────────────────────────────────────────────────┐
│  V4 活动层（实例，HRBP发起）                                       │
│  考核活动 → 方案快照 + 活动级覆盖配置                              │
└─────────────────────────────────────────────────────────────────┘
```

## 配置更新机制

### 实时更新
- 基础配置（V1）：修改后立即生效，所有引用处同步更新
- 方案配置（V2）草稿：修改后立即生效，版本号自动+1
- 活动配置（V4）通知/未开始环节：修改后立即生效

### 版本管理
- 基础配置（V1）被已启用方案引用时：读取快照值
- 方案配置（V2）已启用：修改后版本号+1，已进行活动使用原快照
- 活动快照（scheme_snapshot）：生成后不可变

### 快照不可变
- 活动快照：活动启动时从方案 copy 全部配置，执行中不可修改
- 历史数据：明细表支持 active/voided 状态，不可修改

## 核心概念

### 流程引擎
- **流程定义**：业务流程的"图纸"，1个定义可有N个版本
- **流程版本**：图纸的修订版本号，允许1个草稿+N个发布版本
- **发布**：将设计好的流程定义激活到引擎，1次发布对应1个版本
- **流程实例**：定义的一次实际运行，遵循"版本隔离"原则
- **表单定义**：定义业务数据的字段构成、校验规则、布局样式

### 流程与表单关系
- 模式1：流程驱动表单（标准审批流）
- 模式2：表单驱动流程（动态表单、自定义流程）

### 状态位
三要素：状态、事件（转移条件）、动作
- 审批中 → 通过 / 不通过 / 已作废 / 已驳回 / 已撤回

## 目标用户

### 流程引擎
| 角色 | 场景 |
|------|------|
| 系统管理员(HRIS) | 流程引擎配置 |
| 超级用户(HRIS) | 全公司流程查看、异常干预 |
| 员工 | 提交申请、参与绩效 |
| 经理/主管 | 审批下属申请 |
| 流程审批人 | 审批特定类型申请 |

### 绩效管理
| 角色 | 场景 |
|------|------|
| 系统管理员 | 绩效基础信息配置 |
| COE | 制定绩效考核规则 |
| HRBP | 开启绩效考核活动、跟进进展 |
| 被考核人 | 目标制定、自评、结果确认、面谈、申诉 |
| 考核人 | 绩效评分 |
| 审定人 | 强制分布调整 |
| 审批人 | 审批目标/评价/审定/面谈结果 |

## 已识别的问题
- OEHR 历史绩效流程硬编码过多，无法适配灵活配置需求
- 需要通过流程引擎+配置化重构绩效产品

# 流程引擎需求清单

**目的**：绩效系统对流程引擎的全部需求，供流程引擎 PM 核对开发。
**范围**：仅列出绩效业务需要的流程能力，不涉及引擎自身设计、排期、技术实现。
**更新**：2026-06-30

---

## 一、流程定义（7 条流程）

绩效系统需要流程引擎支持以下 7 条预置流程。每条流程的节点顺序、审批人规则、驳回规则、跳过规则已确认。

### 1.1 通用 PBC 目标制定流程

```
开始 → 目标制定（被考核人）→ 间接上级审批（可跳过）→ 直接上级审批 → 结束
```

| 节点 | 处理人 | 操作 | 驳回 |
|------|--------|------|------|
| 目标制定 | 被考核人（表单字段） | 填写+提交 | — |
| 间接上级审批 | 间接上级（汇报关系） | 同意/不同意/转交 | 可驳回到发起人 |
| 直接上级审批 | 直接上级（汇报关系） | 同意/不同意/转交 | 可驳回到发起人 |

**跳过规则**：间接上级/直接上级为空时自动跳过
**审批方式**：整表审批，非逐条目标审批

### 1.2 线上导入目标制定流程

```
开始 → 目标导入（HRBP批量）→ 员工确认 → 间接上级审批（可跳过）→ 直接上级审批 → 结束
```

| 节点 | 处理人 | 操作 | 驳回 |
|------|--------|------|------|
| 目标导入 | HRBP（组织角色） | 上传Excel+提交（批量处理） | — |
| 员工确认 | 被考核人（表单字段） | 同意/不同意 | 不同意退回 HRBP 重新导入 |
| 间接上级审批 | 间接上级（汇报关系） | 同意/不同意/转交 | 可驳回到发起人 |
| 直接上级审批 | 直接上级（汇报关系） | 同意/不同意/转交 | 可驳回到发起人 |

**特殊需求**：节点 1 为批量处理节点，多人同时流入 HRBP，需引擎支持：
- 业务层识别批量节点，呈现批量处理页面（非单人表单）
- 批量节点通知平衡：避免重复提醒 + 保证新待办可感知
- 按行校验：某行失败不影响其他行提交

### 1.3 线上考核流程

```
开始 → 被考核人自评 → 间接上级评分（可跳过）→ 直接上级评分 → 结束
```

| 节点 | 处理人 | 操作 | 驳回 |
|------|--------|------|------|
| 被考核人自评 | 被考核人（表单字段） | 填写完成值+自评+提交 | — |
| 间接上级评分 | 间接上级（汇报关系） | 评分+提交 | 可驳回到发起人 |
| 直接上级评分 | 直接上级（汇报关系） | 评分+提交 | 可驳回到发起人 |

**驳回规则**：驳回后全部 active 行 voided，全流程重来
**字段权限**：上级不可修改员工填写的完成值/完成情况，仅填写指标评价、总评语、初评等级

### 1.4 结果审定流程 — 汇报线逐级模式

```
开始 → 直接上级评级 → 间接上级评级（可跳过）→ 逐级上级评级（到停止人员结束）→ HRBP审批（仅远超预期）→ 结束
```

| 节点 | 处理人 | 操作 | 条件 |
|------|--------|------|------|
| 直接上级评级 | 直接上级（汇报关系） | 调整等级+提交 | 必过 |
| 间接上级评级 | 间接上级（汇报关系） | 调整等级+提交 | 为空时跳过 |
| 逐级上级评级 | 逐级上级（汇报线） | 调整等级+提交 | 到停止人员（用户配置）结束 |
| HRBP审批 | HRBP（组织角色） | 同意/不同意 | 仅等级=远超预期时流入 |

**审定模式选择**：方案配置时由管理员选择（汇报线逐级/组织架构逐级），活动启动后不可变

### 1.5 结果审定流程 — 组织架构逐级模式

与汇报线逐级相同，差异仅在：
- 逐级上级来源为**组织架构**（非汇报关系）
- 无停止人员规则，到顶层自动结束

### 1.6 结果确认流程

```
开始 → 本人确认 → [同意] → 结束
                  → [不同意] → HRBP知悉 → 结束
```

| 节点 | 处理人 | 操作 | 说明 |
|------|--------|------|------|
| 本人确认 | 被考核人（表单字段） | 同意/不同意 | 不同意时必填原因 |
| HRBP知悉 | HRBP（组织角色） | 知悉 | 知悉后流程结束，争议线下处理 |

**终态不可逆**：结果确认不支持驳回，不同意进入 HRBP 知悉后终态不可逆
**顺序约束**：确认永远是最后一个环节（面谈必须在确认前完成）

### 1.7 绩效面谈流程

```
开始 → 本人填写 → 直接上级审批 → 结束
```

| 节点 | 处理人 | 操作 | 驳回 |
|------|--------|------|------|
| 本人填写 | 被考核人（表单字段） | 填写面谈记录+提交 | — |
| 直接上级审批 | 直接上级（汇报关系） | 同意/不同意 | 可驳回到发起人 |

## 二、审批操作需求

绩效系统所有流程节点需要以下审批操作：

| 操作 | 需求 | 说明 |
|------|------|------|
| 提交 | ✅ | 原子操作：校验→计算→存储→提交流程 |
| 同意 | ✅ | 审批通过，流转到下一节点 |
| 不同意 | ✅ | 审批不通过，需填写原因 |
| 驳回 | ✅ | 发起人外所有节点均可驳回 |
| 转交 | ✅ | 将当前审批任务转交他人 |
| 撤回 | ✅ | 发起人在下一节点处理前可撤回 |
| 催办 | ✅ | 手动/自动发送催办通知 |
| 加签 | ✅ | 审批过程中增加审批人 |

**不涉及**：驳回到上一步、驳回到历史节点、跳转、作废（引擎标准能力，绩效业务不特殊要求）

---

## 三、审批人规则

### 3.1 审批人来源

| 类型 | 来源 | 说明 |
|------|------|------|
| 表单字段 | 被考核人本人 | 发起节点 |
| 汇报关系 | 直接上级/间接上级 | 按被考核人的汇报线查找 |
| 组织角色 | HRBP | 按被考核人所在部门查找 |
| 组织架构 | 逐级上级（组织架构模式） | 按被考核人所在组织层级查找 |
| 用户配置 | 停止人员 | 审定流程中用户手动输入 |

### 3.2 核心规则

- **单人审批**：所有节点均为单人审批（系统设计只有一个人或为空）
- **为空跳过**：间接上级/直接上级/HRBP 为空时，自动跳过该节点
- **身份重复跳过**：处理人身份重复时跳过（如直接上级=间接上级）
- **无多人审批场景**：不存在同一节点多人审批的需求

## 四、批量处理需求

仅目标制定线上导入流程（flow_goal_setting_import）涉及批量处理：

| 需求 | 说明 |
|------|------|
| 批量处理节点 | 多人同时流入 HRBP，引擎需识别并提供批量处理页面 |
| 按行校验 | 批量导入按行校验、按行成功/失败，单行失败不影响其他行 |
| 批量通知 | 批量节点持续流入新人员时，避免重复提醒，同时保证新待办可感知 |
| 批量表单 | HRBP 使用批量导入表单页面（Excel 上传+预览），非标准单人审批表单 |

---

## 五、消息通知需求

| 通知类型 | 触发时机 | 接收人 |
|---------|---------|--------|
| 待办通知 | 新节点流入时 | 节点处理人 |
| 催办通知 | 手动催办 / 超时自动催办 | 当前节点处理人 |
| 超时提醒 | 节点超过方案配置的截止时间 | 当前节点处理人 + HRBP |
| 驳回通知 | 审批被驳回时 | 被驳回节点处理人 |
| 结果通知 | 流程结束/确认完成时 | 被考核人 |

**通知渠道**：站内信 + 飞书（引擎 V3 规划对接）

## 六、流程实例管理需求

绩效 V4 考核活动需要以下流程实例管理能力：

| 能力 | 说明 |
|------|------|
| 流程实例监控 | 查看活动下所有流程实例的状态、当前节点、处理人 |
| 环节进度跟踪 | 按考核组/部门/环节查看完成进度 |
| 手动开启/关闭环节 | HRBP 手动控制环节启停 |
| 批量操作 | 批量开启/关闭/暂停/恢复 |
| 强制终止 | HRBP 强制终止个别或批量流程实例（含原因记录） |
| 流程退回 | 退回至上一节点（含原因） |
| 流程转交 | 将当前任务转交他人（含记录） |
| 环节暂停/恢复 | 暂停某环节下所有流程，恢复后继续 |
| 流程挂起/恢复 | 挂起单个流程实例，恢复后继续 |

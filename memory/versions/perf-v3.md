## 本次确认的文档方向

- V3 PRD 按产品经理视角书写，主干以业务场景、用户角色、用户任务、字段规则、异常提醒、流程状态、验收标准为主。
- 原型仅作为交互参考，不作为业务逻辑唯一依据；业务规则以 PRD 为准。
- 主文档不写代码、接口入参出参、数据库表结构、锁机制等技术实现细节。

## 已确认的核心业务规则

- 审批是整表审批，不存在单条目标或单条指标审批。
- 每次提交、每次审批都是独立业务记录；审批动作不直接改写员工提交的业务数据。
- 目标制定审批通过后目标生效；驳回后退回对应节点，员工或节点处理人重新提交新记录。
- 所有审批节点可以驳回，首节点不可驳回；驳回按流程模板或逐级规则处理。
- 统一使用”直线上级””双线上级””HRBP”。
- 量化指标由员工填写完成值，系统按配置的计算规则计算得分；缺少计算规则时阻断提交。
- 定性指标由员工填写完成情况并选择评价等级，系统按评价规则等级对应分值取分。
- 双线上级/直线上级不可修改员工填写的完成值、完成情况和评价结果；仅填写指标评价、总评语、初评等级。
- 强制分布按方案配置计算；强控超限阻断，弱控超限允许提交但提示。
- 跳过审定或强制结束使用方案配置的默认等级；如果只是跳过单个节点，后续审定节点仍可调整。
- 结果确认不支持驳回，终态不可逆；员工不同意后进入流程模板配置的后续处理分支。
- 批量导入（目标/考核结果）由 HRBP 执行，提交后进入直线上级审批，通过后生效；员工不参与确认或审批。
- 批量导入按行校验、按行成功/失败，某一行失败不影响其他成功行提交。
- 考核结果批量导入为独立线上导入流程，不走员工自评节点。

# 绩效管理 V2：绩效方案

## 版本目标
配置绩效方案，定义考核组和考核环节设计。

## 功能清单（28项）

### 绩效方案
- 入口：GHR / 绩效管理 / 绩效方案 / 绩效方案
- 操作：编辑、复制、启用/停用、删除
- 方案详情（只读）+ 关联活动

### 方案配置目录
步骤：基本信息 → 考核组 → 考核环节设计

**配置模式**：合并配置（全局共用）/ 按考核组配置（各组独立）
- 模板：月度绩效、半年度绩效、年度绩效、自定义（含月度/年度场景快速选择）
- 配置进度：完成度百分比

#### 基本信息
- 名称、方案版本（创建v1，更新自动递增）、周期类型（月度/半年度/年度/自定义）、是否启用、动态考核、绩效等级

#### 考核组
- 模式选择
- 规则：按条件选择（部门/职位/职级）+ 按人员选择
- 排除人员配置

#### 考核环节设计

| 环节 | 关键配置 |
|------|----------|
| 目标制定 | 开启方式（手动/自动）、结束方式（手动/超时自动结束）、流程模板、强制结束规则 |
| 绩效考核 | **评价规则**（引用V1，必填）、开启方式、结束方式、流程模板、节点权重分配、强制结束规则 |
| 结果审定 | 开启方式、结束方式、**强制分布规则**（单选标准/宽松/严格）、**控制模式**（强控/弱控）、等级配比详情、强制结束规则 |
| 结果确认 | 开启方式（手动/自动）、结束方式（手动/超时自动结束）、通知、流程模板（**无强制结束规则**） |
| 绩效面谈 | 开启方式、结束方式、流程模板、强制结束规则 |

### 绩效表单
- 目标制定表单：基本信息、模块信息、目标/指标信息
- 绩效考核表单：批量评分、批量导入
- 绩效审定表单
- 员工确认表单
- 表单字段/流程业务信息参考：GHR绩效模块业务表单、流程收集

预置表单类型：
| 表单 | 用途 |
|------|------|
| PBC目标制定表单 | 目标制定环节，员工/上级填写绩效目标 |
| 绩效考核表单 | 绩效考核环节，考核人评分 |
| 结果审定表单 | 结果审定环节，审定人调整等级 |
| 员工确认表单 | 结果确认环节，员工确认绩效结果 |

### 绩效流程
- 正向操作：提交（通过）、转交
- 审定模式（汇报线/组织架构）、停止人员、HRBP知悉节点 → 均在**流程配置**中定义，方案只选择用哪个流程

## 方案与活动关系

```
方案（V2配置）               活动（V4执行）
┌──────────────────┐       ┌──────────────────┐
│ 考核组            │ copy→│ 参与人员（可微调） │
│ 环节配置          │ copy→│ 环节配置（可修改） │
│ 流程模板          │ copy→│ 流程模板          │
│ 表单模板          │ copy→│ 表单模板          │
│ 强制分布          │ copy→│ 强制分布（可修改） │
│ 评价规则          │ copy→│ 评价规则          │
│ 节点权重          │ copy→│ 节点权重（可修改） │
└──────────────────┘       └──────────────────┘
                                 │
                                 ▼
                           最终按活动配置执行
```

- 活动实例化时 copy 方案全部配置
- 活动可修改，执行时只看活动配置
- 方案停用后，已发起的活动不受影响

## 关键设计决策
- 考核组支持多组，按条件+人员交集确定范围
- 强制分布规则在结果审定环节配置，支持标准/宽松/严格三种预设 + 强控/弱控
- 评价规则从考核组移至绩效考核环节统一管理
- 方案有版本号概念：创建时v1，每次更新自动递增
- 配置模式支持合并配置（全局共用）和按考核组配置（各组独立）
- 结果确认环节无强制结束规则，其余4个环节均有
- 流程模板关联流程引擎中的流程模型
- 绩效申诉在V2标记为待定（删除线）
- 角色权限由组织人事模块维护，绩效不单独维护
- 消息通知后续版本统一做

## 字体规范

- 正文字体: Inter, Noto Sans SC, -apple-system, BlinkMacSystemFont, sans-serif
- 等宽字体: JetBrains Mono, SF Mono, monospace（数据展示）
- 页面标题: 18px, c9, font-weight: 700
- 卡片标题: 14px, c8, font-weight: 600
- 正文字号: 13-14px, c7
- 辅助文字: 11-12px, c5

## 导航层级结构

```
┌─────────────────────────────────────────────────────┐
│  Header (52px)                                      │
│  [Logo] [全部菜单按钮] [搜索框 ⌘K] [通知][帮助][头像] │
└─────────────────────────────────────────────────────┘
         ↓ 点击"全部菜单"
┌─────────────────────────────────────────────────────┐
│  Mega Menu（全屏覆盖，带遮罩）                       │
│  模块列表 → 分组网格 → 点击跳转                      │
└─────────────────────────────────────────────────────┘
         ↓ 点击分组/功能项
┌──────────┬──────────────────────────────────────────┐
│ Sidebar  │  Content                                │
│ (240px)  │  [页面标题 + 操作按钮]                   │
│ 模块头部  │  [统计卡片 / Tab / 列表 / 空状态]        │
│ 分组列表  │                                          │
│ 功能项   │                                          │
└──────────┴──────────────────────────────────────────┘
```

## 导航交互规则

### Mega Menu

| 操作 | 行为 |
|------|------|
| 点击「全部菜单」 | 按钮高亮(.on) + 遮罩出现 + Mega Menu 从上方滑入 |
| 点击遮罩 / 再次点击按钮 | 关闭 Mega Menu |
| 点击分组 | 关闭菜单 → 切换侧边栏 → 跳转该分组首项 |

### 侧边栏

| 操作 | 行为 |
|------|------|
| 点击分组标题 | 展开/折叠该分组（toggle .on） |
| 展开箭头 | 选中时旋转 90°（transform: rotate(90deg)） |
| 点击功能项 | 该项高亮：蓝色背景 + 左侧 3px 指示条 + font-weight: 600 |

### 搜索框

- 居中，宽度 320px
- 默认背景 c0，聚焦时显示蓝色边框
- 快捷键提示 ⌘K

## 组件规范

### 按钮

| 类型 | 背景 | 文字 | 边框 | Hover |
|------|------|------|------|-------|
| 主按钮 (.btn-p) | #2563EB | #FFF | 无 | #1D4ED8 |
| 次按钮 (.btn-s) | #FFF | c6 | 1px solid c2 | c3, c8, c0 |

- 高度: 32px
- 圆角: 6px
- 内边距: 0 14px

### 统计卡片

- 顶部 3px 纯色条（不同颜色区分）
- Hover: translateY(-2px) + 阴影增强
- 数值: 22px JetBrains Mono, font-weight: 700

### Tab 栏

- 背景 c0
- 选中态: 底部 2px 主色边框 + 蓝色文字 + font-weight: 600
- 计数徽章: 选中时背景 pr50, 文字 pr

### 任务列表

- 行间 1px 分割线
- Hover: 背景 c0, 标题变 pr 色
- 优先级条: 左侧 3px 纯色（h=红/m=黄/l=灰）
- 状态标签: 待审批(ac50)/审批中(pr50)/已通过(em50)

## 组件规范（纯HTML）

- 不使用任何UI框架（无 Ant Design / Element UI）
- 自定义组件通过 HTML + CSS + JS 实现
- Mock数据内联在HTML文件中
- 页面间通过链接或JS切换

## 关键约定

1. 去掉"绩效模板"，配置全在绩效方案里完成
2. Mock数据统一管理，便于替换真实API
3. 每个版本独立演示，不依赖未完成功能
4. 废弃所有旧 React 原型，全部重新输出纯 HTML
